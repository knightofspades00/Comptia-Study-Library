Core 1 (220-1101)

📱 1.0 — Mobile Devices 15%

▼

Laptop Hardware Components

Laptops have many proprietary components. Key items to know:

  • Display types: TN (fast, poor viewing angles), IPS (best color/angles), OLED (deep blacks, power-efficient on dark content), VA (good contrast)
  • Storage: SATA SSD, NVMe M.2 (PCIe x4), eMMC (soldered, budget devices)
  • Memory: SO-DIMM (laptop DDR4/DDR5), many modern laptops have soldered RAM
  • Battery: Li-ion / Li-polymer. Measured in mAh or Wh. Replace when capacity drops below ~80%
  • Thermal paste: Applied between CPU and heat sink/fan assembly. Dries out over time causing thermal throttling

Mobile OS Comparisons

iOS: Closed ecosystem, App Store only, tighter security, no sideloading (without MDM). Android: Open source (AOSP), sideloading allowed, more fragmentation. Both support MDM, VPN, remote wipe.

⚠️ Exam tip: Know the difference between iOS and Android app sources. iOS = App Store only. Android = Play Store + APK sideloading.
💼 Real-world — the lint-in-the-charging-port fix

The #1 most common "my phone won't charge" support ticket isn't a bad cable or a dead battery — it's pocket lint packed into the charging port. Take a flashlight, look in the port; if you see fuzz, gently scrape it out with a wooden toothpick (never metal). This single trick fixes ~50% of "won't charge" complaints and saves real money on unnecessary repairs.

🌐 2.0 — Networking 20%

▼

TCP/IP Basics

The A+ exam tests practical networking, not deep theory. Focus on:

  • IP addressing: IPv4 (32-bit, dotted decimal), IPv6 (128-bit, hex colon notation)
  • Subnet masks: /24 = 255.255.255.0 (256 hosts, 254 usable), /16 = 255.255.0.0, /8 = 255.0.0.0
  • Private IP ranges: 10.x.x.x, 172.16-31.x.x, 192.168.x.x
  • APIPA: 169.254.x.x — assigned when DHCP fails. Sign of a connectivity problem
  • Loopback: 127.0.0.1 — always refers to local machine

📌 Exam tip — ports show up everywhere

Memorize these. CompTIA combines port knowledge into scenario questions: "A user can browse HTTP sites but not HTTPS — which port should you check?" → 443. "Email arrives but won't send" → SMTP / port 25 (or 587 for secure). Two-letter pairs that often confuse: POP3 (110) vs IMAP (143) — POP3 downloads-and-deletes, IMAP keeps a synced copy on the server.

Network Devices

  • Hub: Layer 1, broadcasts to all ports — legacy, avoid
  • Switch: Layer 2, uses MAC addresses, sends frames only to destination port
  • Router: Layer 3, uses IP addresses, connects different networks
  • WAP: Wireless access point, extends wired network wirelessly
  • Firewall: Filters traffic by rules; hardware or software
  • Modem: Converts analog (cable/DSL) signal to digital

🔧 3.0 — Hardware 25%

▼

Motherboard Components

  • CPU socket types: Intel LGA (pins on motherboard), AMD AM5/PGA (pins on CPU)
  • Chipset: Northbridge (fast: CPU-RAM-GPU) and Southbridge (slow: storage, USB, audio) — modern CPUs integrate northbridge
  • RAM slots: DIMM (desktop), SO-DIMM (laptop). Install in matching slots for dual-channel
  • PCIe slots: x16 (GPU), x8, x4, x1 (NIC, sound cards)
  • Storage interfaces: SATA III (6 Gbps), M.2 NVMe (up to 32 Gbps PCIe 4.0 x4)
  • BIOS/UEFI: Firmware that initializes hardware on boot. UEFI supports GPT, Secure Boot, faster POST

RAM Types

TypeSpeed RangeNotes
DDR42133–3200+ MHz288-pin DIMM, most common current desktop/laptop
DDR54800–8000+ MHz288-pin DIMM, newer platforms
ECC RAMAny DDR genError-correcting, used in servers

Storage Technologies

  • HDD: Spinning platters, 5400/7200 RPM, high capacity, low cost per GB, slow random access
  • SSD (SATA): Flash-based, 550/500 MB/s read/write, 2.5" or M.2 SATA
  • NVMe SSD: PCIe interface, 3500+ MB/s, much faster than SATA SSD
  • RAID 0: Striping, speed, no redundancy. RAID 1: Mirroring, redundancy. RAID 5: Striping + parity, needs 3+ drives, can lose 1. RAID 10: Mirror + stripe, needs 4 drives
💡 PSU: 80 PLUS rating (Bronze, Silver, Gold, Platinum, Titanium) indicates efficiency. Modular PSUs allow removing unused cables.

☁️ 4.0 — Virtualization & Cloud 11%

▼

Cloud Service Models

  • IaaS (Infrastructure as a Service): Raw compute, storage, network — AWS EC2, Azure VMs. You manage OS up.
  • PaaS (Platform as a Service): Managed runtime environment — Heroku, Azure App Service. You manage app & data only.
  • SaaS (Software as a Service): Ready-to-use app — Gmail, Office 365, Salesforce. Provider manages everything.

Cloud Deployment Types

  • Public: Shared infrastructure (AWS, Azure, GCP)
  • Private: Dedicated to one org, on-prem or hosted
  • Hybrid: Mix of public + private, connected
  • Community: Shared by orgs with common needs (gov, healthcare)

Virtualization Basics

  • Hypervisor Type 1: Bare-metal (VMware ESXi, Hyper-V, KVM) — runs directly on hardware
  • Hypervisor Type 2: Hosted (VirtualBox, VMware Workstation) — runs as app on OS
  • VM vs Container: VMs virtualize full hardware; containers share host OS kernel (Docker)
📌 Exam tip — Type 1 vs Type 2 hypervisors

If the question describes a data center, production server, or enterprise virtualization → Type 1 (bare-metal). If it describes a laptop, student lab, or "test this on your own PC" → Type 2 (hosted). The keyword "directly on the hardware" always points to Type 1.

Windows Editions

EditionKey Features
HomeConsumer, no domain join, no Group Policy, no BitLocker (no drive encryption)
ProDomain join, Group Policy, BitLocker, Hyper-V, Remote Desktop
EnterprisePro + AppLocker, DirectAccess, Windows To Go, volume licensing

Key Windows Tools & Commands

  • ipconfig — show IP config; /release, /renew for DHCP; /flushdns
  • ping — test reachability
  • tracert / traceroute — trace route hops
  • netstat — active connections and listening ports
  • nslookup — DNS query tool
  • chkdsk — check/repair filesystem errors
  • sfc /scannow — System File Checker
  • DISM — Deployment Image Servicing and Management (repair Windows image)
  • msconfig — startup/services configuration
  • regedit — Windows Registry Editor
  • gpupdate /force — force Group Policy refresh
  • diskpart — CLI disk partitioning
📌 Exam tip — sfc vs chkdsk vs DISM

All three "fix Windows" commands, but they fix different things:
• sfc /scannow = repairs corrupted system files (DLLs, EXEs)
• chkdsk /f = repairs disk/file-system errors (bad sectors, lost clusters)
• DISM = repairs the underlying Windows component store when sfc can't finish
Order when troubleshooting: sfc first → if it can't fix, run DISM /Online /Cleanup-Image /RestoreHealth → then sfc again.

💼 Real-world — try these now

Open Command Prompt (Win+R, type cmd) and try ipconfig /all — that's every networking setting on your computer. Then ping google.com. These are the first two commands a help-desk tech runs when a user says "internet isn't working." Knowing them cold is a job-day-one skill.

Windows File Systems

  • NTFS: Default for Windows drives. Supports permissions, encryption (EFS), journaling, large files
  • FAT32: Universal compatibility, max 4 GB per file, max 32 GB volume (Windows format)
  • exFAT: Flash drives, max 16 EB file size, good cross-platform

macOS & Linux Basics

  • macOS uses HFS+/APFS, Terminal with bash/zsh, Spotlight search
  • Linux: Everything is a file, case-sensitive filenames, root is /, uses package managers (apt, yum)
  • Key Linux commands: ls cd pwd chmod chown grep ps sudo

🔐 2.0 — Security 25%

▼

Malware Types

  • Virus: Attaches to files, requires user action to spread
  • Worm: Self-replicates across network without user action
  • Trojan: Disguised as legitimate software
  • Ransomware: Encrypts files, demands payment
  • Spyware: Monitors user activity, steals data
  • Rootkit: Hides in OS, very difficult to detect/remove. May require offline scan
  • Keylogger: Records keystrokes
  • Botnet/Zombie: Compromised machine controlled remotely for DDoS, spam

Social Engineering

  • Phishing: Bulk fake email; Spear phishing: targeted; Whaling: targeting executives
  • Vishing: Voice/phone phishing
  • Smishing: SMS phishing
  • Tailgating/Piggybacking: Following someone through a secure door
  • Shoulder surfing: Watching someone enter credentials
  • Dumpster diving: Finding sensitive info in trash
📌 Exam tip — "BEST" answer pattern

When a question asks for the BEST response (often after "User clicks a suspicious link"), CompTIA expects: disconnect the device from the network first (containment), then scan/remove, then re-image if needed. Pulling the plug is the cheap, correct first step — it stops lateral spread.

Windows Security Features

  • BitLocker: Full-disk encryption. Requires TPM 1.2+ or USB key. Pro/Enterprise only
  • Windows Defender: Built-in AV/anti-malware
  • Windows Firewall: Host-based packet filtering
  • UAC: User Account Control — prompts before elevated actions
  • EFS: Encrypting File System — per-file/folder encryption (NTFS only)

🔧 5.0 — Hardware & Network Troubleshooting (Core 1) 29%

▼

Troubleshooting Methodology (Always Follow This Order)

  1. Identify the problem — gather info, question user, check logs, duplicate if possible
  2. Establish a theory of probable cause — consider multiple causes (start simple)
  3. Test the theory — if confirmed, move on; if not, re-theorize or escalate
  4. Establish a plan of action — consider side-effects, schedule downtime if needed
  5. Implement the solution or escalate — apply fix, or refer to next tier
  6. Verify full system functionality — test, and implement preventive measures
  7. Document findings, actions, and outcomes — update ticket, knowledge base
📌 Exam tip — "FIRST step" always means step 1

If a question asks "What is the FIRST step?" — the answer is almost always Identify the problem. CompTIA is checking that you don't skip ahead to "Replace the part" or "Reinstall Windows" without diagnosing first. When unsure, pick the option about asking the user or checking what changed.

🎯 Mnemonic — 6 steps

"I'll Try To Plan In Very Detailed-ways" = Identify → Theory → Test → Plan → Implement → Verify → Document. CompTIA technically labels Verify & Document as separate, but most resources count it as 6 steps with documentation rolled into the last.

Power Issues

  • No power at all: Check outlet, PSU switch, power cable, surge protector. Try known-good PSU.
  • Intermittent shutdowns: Overheating (clean fans/vents, reapply thermal paste), failing PSU, bad capacitors on motherboard
  • Burning smell: Immediately power off. Check for bulging/leaking capacitors, scorched components
  • POST beep codes: Vary by BIOS vendor. Generally: 1 short = OK, 1 long + 2-3 short = GPU/RAM issue. Always check mobo manual.
  • POST errors on screen: "Boot device not found" = storage issue or wrong boot order. "CMOS checksum error" = replace CMOS battery (CR2032)

Display Troubleshooting

  • No image, PC on: Check cable connections, try different port (HDMI→DP), test with another monitor. If iGPU: reseat RAM
  • Dim/dark screen: If image visible with flashlight → backlight failed (CCFL or LED inverter). If laptop → check Fn+brightness keys first
  • Flickering: Loose cable, failing backlight, refresh rate mismatch, GPU driver issue
  • Dead pixels: Always-black = stuck off (dead). Always-lit = stuck on. Small clusters often warranty-covered
  • Color distortion / burn-in: GPU output issue or panel degradation. Run pixel-refresh if OLED
  • VGA fuzzy: Analog signal — tighten connector, increase resolution
  • Overscan on TV: Adjust display scaling in GPU driver settings or TV picture settings

Storage Troubleshooting

  • HDD clicking/grinding: Mechanical failure imminent — back up immediately, replace drive
  • Drive not detected in BIOS: Check data cable, power cable, try different SATA port. For M.2: reseat card
  • Slow read/write: HDD fragmented (run defrag), SSD nearly full (>90%), SATA in wrong mode (IDE vs AHCI)
  • SMART errors: Self-Monitoring Analysis and Reporting Technology. Warning = drive failure likely. Back up and replace
  • Bad sectors: Run chkdsk /r to find and mark bad sectors. Frequent bad sectors = replace drive
  • OS not found / boot failure: Run bootrec /fixmbr, bootrec /fixboot, bootrec /rebuildbcd from WinRE

RAM Troubleshooting

  • POST beep codes or no boot: Reseat RAM, try one stick at a time, test with known-good stick
  • Random BSODs / crashes: Often RAM failure. Run Windows Memory Diagnostic (mdsched.exe) or MemTest86
  • System won't boot with new RAM: Incompatible speed/type, needs XMP profile enabled in BIOS, or slots need to be dual-channel paired correctly

Printer Troubleshooting

  • Laser — ghost images / light shadows: Fuser assembly worn, or drum not cleaning properly. Replace fuser or drum unit
  • Laser — smeared/smudged toner: Fuser not reaching proper temperature. Replace fuser
  • Laser — streaks/lines on page: Scratched drum, toner cartridge low or damaged
  • Laser — blank pages: Toner cartridge empty or sealing tape not removed (new cartridge)
  • Inkjet — streaky/missing colors: Clogged nozzle — run print head cleaning utility
  • Inkjet — colors incorrect: Wrong color profile, wrong paper type selected, low ink in specific cartridge
  • Paper jam: Follow printer jam-clearing procedure, never force paper. Check for torn bits left inside
  • Printer offline: Check network/USB connection, restart print spooler service (services.msc → Print Spooler → Restart)
  • Incorrect paper size error: Verify paper in tray matches driver settings
  • Thermal printer — no output: Paper loaded backwards (thermal side must face print head). Check thermal head for debris

Core 2 (220-1102) — Additional Domains

🛠️ 3.0 — Software Troubleshooting (Core 2) 22%

▼

Windows Boot Issues

  • BSOD (Blue Screen of Death): Stop error with code. Common causes: bad drivers, RAM, overheating, corrupted OS. Note the stop code (e.g., MEMORY_MANAGEMENT, IRQL_NOT_LESS_OR_EQUAL)
  • Spinning wheel / won't finish loading: Corrupted startup files, failed update, bad driver. Boot to WinRE → Startup Repair
  • Windows boot options:
    • F8 (legacy) or Shift+Restart → Safe Mode — loads minimal drivers only
    • Safe Mode with Networking — adds network drivers
    • Safe Mode with Command Prompt — no GUI
    • WinRE (Windows Recovery Environment) — Startup Repair, System Restore, Command Prompt, Reset this PC, System Image Recovery
  • Boot loop: Stuck in restart cycle. Often after failed update. Boot to WinRE → Uninstall Updates or use System Restore
  • Inaccessible Boot Device BSOD: Storage driver failure (often after SATA mode change IDE↔AHCI), corrupted boot sector
  • Missing NTLDR / BOOTMGR: Run bootrec /fixmbr and bootrec /rebuildbcd

Application Issues

  • Application won't launch: Check compatibility mode (right-click → Properties → Compatibility), missing .NET/VC++ runtime, corrupted install
  • Application crashes frequently: Check Event Viewer → Windows Logs → Application for error codes. Reinstall or update app
  • Application not responding (hung): Task Manager → right-click → End Task. If recurring, check RAM, run sfc /scannow
  • High CPU/RAM usage: Task Manager → identify culprit process. Could be malware, runaway service, or memory leak
  • Application won't uninstall: Use Settings → Apps, or third-party uninstaller. Manually remove registry entries if needed
  • Missing DLL errors: Reinstall the associated application, or install correct Visual C++ Redistributable/DirectX runtime
  • Compatibility issues (older apps on Win 10/11): Run as Administrator, enable compatibility mode for older Windows version

Malware Removal Procedure

  1. Identify malware symptoms (slow, popups, browser hijacked, AV disabled)
  2. Quarantine / isolate the infected machine from the network
  3. Boot to Safe Mode to prevent malware from loading
  4. Run updated malware scanner (Malwarebytes, Windows Defender Offline)
  5. Remove identified malware per tool instructions
  6. Remediate — restore affected files, fix browser settings, re-enable security
  7. Schedule scans and update software — patch OS and applications
  8. Educate user on how infection occurred
💡 Rootkit note: If a rootkit is suspected, offline scanning (bootable AV media) is required — the rootkit hides from a running OS scan.
🎯 Mnemonic — malware removal order (7 steps)

"I Quickly Scan Remove, Repair, Schedule, Educate" = Identify → Quarantine → Safe mode → Remove → Remediate → Schedule scans → Educate. Note: Quarantine comes BEFORE running the scan — disconnect first so malware can't spread or update itself during the scan.

Browser Issues

  • Browser hijack: Homepage/search engine changed without permission. Remove suspicious extensions, reset browser settings, run malware scan
  • Pop-up ads everywhere: Adware installed. Remove via Programs and Features + malware scan
  • Certificate errors: Expired/untrusted cert, wrong system date/time, or MITM attack
  • Slow browsing: Too many extensions, full cache, DNS issue. Clear cache, disable extensions, try different DNS (8.8.8.8)

Mobile Device Software Troubleshooting

  • App crashing: Force stop → clear cache → uninstall/reinstall
  • Slow performance: Restart device, clear app cache, free storage space (<10% free = very slow)
  • Battery draining fast: Check battery usage in settings — identify rogue app. Disable background app refresh
  • Cannot sync: Check credentials, verify Wi-Fi, toggle airplane mode, re-add account
  • Factory reset: Last resort for persistent issues. Wipes all user data — back up first
  • Screen unresponsive: Force restart (model-specific button combination). If persistent → hardware issue (digitizer)

📋 4.0 — Operational Procedures (Core 2) 22%

▼

Documentation Best Practices

  • Network diagrams: Logical (shows IP scheme, VLANs) and Physical (shows cable runs, rack layout)
  • SOPs: Standard Operating Procedures — step-by-step instructions for recurring tasks
  • Knowledge base: Searchable repository of known issues and resolutions
  • Ticketing system: Tracks incidents, service requests, changes. Fields: priority, category, assignee, SLA, status
  • Asset inventory: Hardware/software list with serial numbers, warranty info, license keys
  • Change log: Records all changes made to systems — essential for troubleshooting

Change Management

  • Purpose: Ensure changes are tested, approved, and reversible before deployment
  • Change request: Formal request documenting the change, reason, risk, rollback plan
  • Approval process: Change Advisory Board (CAB) reviews and approves
  • Change window: Scheduled low-impact time (often nights/weekends)
  • Backout plan / rollback: How to undo the change if something goes wrong
  • Documentation: Record what was done, when, by whom, and the outcome
💼 Real-world — why CompTIA hammers change management

The single biggest cause of outages in real IT departments is "someone made a change without telling anyone." Change management exists so that when something breaks at 2 AM, the on-call person can look at the change log and know what changed in the last 24 hours — that's almost always the cause. On the exam, the right answer is usually "document the change" or "submit a change request," not "just fix it quickly."

Backup and Recovery

TypeWhat it backs upRestore timeStorage use
FullEverythingFastestMost
IncrementalChanges since last backup (any type)Slowest (needs full + all incrementals)Least
DifferentialChanges since last full backupFast (needs full + one differential)Grows over time
Synthetic fullCombines full + incrementals into new fullFastMedium

3-2-1 Rule: 3 copies of data, on 2 different media types, with 1 offsite copy.

📌 Exam tip — incremental vs differential

The trick: with differential, every backup contains everything since the last full, so it gets bigger every day, but you only need 2 tapes to restore. With incremental, each backup only has changes since any previous backup, so files are smaller but a restore needs the full + every incremental since. CompTIA often asks: "Which has the slowest restore?" → Incremental.

Safety Procedures

  • ESD (Electrostatic Discharge): Can destroy components with static as low as 10V (undetectable by humans). Use ESD wrist strap connected to ground, ESD mat, anti-static bags for components
  • Lifting: Lift with legs not back. Get help or use cart for heavy equipment (>50 lbs)
  • Electrical safety: Never work on live circuits. Disconnect power before opening PSU (capacitors store charge)
  • Toxic materials: Toner is a fine particle — don't inhale. Use mask when cleaning. Laser printer ozone — ensure ventilation. CRT monitors — high-voltage capacitors, don't open
  • Fire suppression: Server rooms use FM-200 or CO₂ (not water). Know the location of fire extinguishers
  • MSDS/SDS: Material Safety Data Sheet — required for hazardous substances. Toner, batteries, cleaning agents
📌 Exam tip — ESD precautions

You can't feel a static shock under 3,000 volts, but a chip can fail at 10 volts. Always assume your hands are charged. The cheap, exam-perfect order: 1) Unplug the system. 2) Wear an ESD wrist strap clipped to a ground point or the chassis. 3) Place components on an ESD mat or in an anti-static bag — not on a regular plastic bag (which generates static).

Remote Access Technologies

  • RDP (Remote Desktop Protocol): Port 3389. Full graphical desktop access to Windows machines. Requires Pro/Enterprise edition
  • VPN: Encrypted tunnel for remote network access. Common types: SSL/TLS (port 443), IPSec
  • SSH: Port 22. Secure CLI access to Linux/network devices
  • Telnet: Port 23. Unencrypted CLI — never use on production systems
  • Remote Assistance (MSRA): Windows tool for helpdesk to view and control user's screen with their permission
  • Third-party tools: TeamViewer, AnyDesk, ConnectWise — used for unattended or attended remote support
  • Screen sharing: macOS Screen Sharing (VNC-based), Google Chrome Remote Desktop

Professionalism

  • Always get proper authorization before accessing someone's system
  • Respect privacy — don't read personal files, avoid unnecessary snooping
  • Set expectations with users — give realistic ETAs, follow up
  • Use plain language with non-technical users; avoid jargon
  • Maintain a positive attitude, don't blame the user
  • Confidentiality — don't discuss client data or issues publicly

📖 Reading / Research Activities

📖 Reading Comprehension

Activity 1: Understanding the POST Process

The Power-On Self-Test (POST) is a firmware-level diagnostic run every time a computer boots. Understanding this process is critical for hardware troubleshooting on the A+ exam.

Read about the POST process in the CompTIA A+ study material, then answer: What is the purpose of POST? What does a single beep vs. multiple beeps indicate? What happens if POST fails to find bootable media? What role does the BIOS/UEFI play after POST completes?
  1. List the six main stages of the POST process in order.
  2. Describe two different POST error codes (beep codes or visual codes) and what they indicate.
  3. Explain the difference between BIOS and UEFI. Why did UEFI replace BIOS as the standard?
  4. A technician powers on a PC and hears 3 long beeps. No display appears. What is the most likely issue and what are the troubleshooting steps?
🔬 Research Activity

Activity 2: Comparing Storage Technologies

Storage technology is a major topic on the A+ Core 1 exam. This activity asks you to research and compare the primary storage interfaces and form factors used in modern computing.

  1. Research and fill in a comparison table of: HDD (SATA), SSD (SATA), NVMe M.2 SSD, and eMMC storage — covering speed, typical use case, price tier, and physical form factor.
  2. Look up the difference between M.2 SATA and M.2 NVMe. Why do both use the same physical slot but perform so differently?
  3. Research the concept of wear leveling in SSDs. Why is this important for longevity?
  4. Find the current average price per GB for HDD vs SATA SSD vs NVMe SSD. What does this tell you about when you'd choose each?

Activity 3: Network Troubleshooting Scenarios

Read the following scenarios carefully, then apply the CompTIA troubleshooting methodology (identify problem → establish theory → test theory → establish plan → implement solution → verify → document) to each.

Scenario A: A user's laptop shows "Limited connectivity" on Wi-Fi. The IP address shown is 169.254.45.12. Other users on the same Wi-Fi are working fine.

Scenario B: A user can browse the web but cannot send or receive email using their desktop email client. Webmail works fine through the browser.

Scenario C: After moving to a new office, a user can't connect to the printer on the network. The printer has an IP of 192.168.1.50. The user's IP is 10.0.0.23/24.
  1. For Scenario A: What does the 169.254.x.x address indicate? What is the most likely root cause? What command would you run first?

  3. For Scenario C: Why can the user not reach the printer? What networking concept explains this, and what solution would fix it?

Activity 4: Security & Operational Procedures Research

The Core 2 exam heavily emphasizes security practices and operational procedures. Research these real-world topics and apply them to the scenarios below.

🖨️ Laser Printer: The 7-Step Imaging Process

One of the most consistently tested A+ topics. Every step happens in order — a question about a symptom (e.g., toner smearing) maps to a specific step. Know them all.

Common Exam Trap
Toner smearing/not fusing = Fuser assembly failure (Step 6). Ghost images (faint duplicate) = Cleaning blade failure (Step 7). Blank pages = Charging or Exposing failure (Steps 2–3).

💾 Storage vs Memory — Quick Rule
"RAM = Remember After Momentarily — it forgets when power is gone."
RAM (volatile) = loses data without power · Storage (HDD/SSD) = keeps data permanently · Cache = fastest, smallest · Virtual memory = pretend RAM using disk space
🔌 Connector Types — Color Coding Trick
"Audio jacks: Green = Go (output/speakers), Pink = Pick up (mic input), Blue = Back (line-in)"
Most PCs follow this color standard. On the exam, if a question says "no sound from speakers," check the green jack first.

⚡ POST Beep Codes — Rule of Thumb
"1 beep = happy, no beep = power problem, many beeps = hardware problem"
1 short beep on BIOS POST = system OK · Continuous beeps = RAM · Repeating short beeps = power supply · Long + short combos = video card (varies by BIOS manufacturer)

🎯 How to Pass the CompTIA A+ (220-1101 / 220-1102)

🗣️ CompTIA Question Language

CompTIA uses specific trigger words that change the right answer. Learn to spot them:

  • "MOST likely" — eliminate impossible answers; pick the best fit
  • "BEST" — there may be 2 correct answers; one is more efficient/secure
  • "FIRST" — follow the correct order/process (e.g., troubleshooting steps)
  • "LEAST" / "EXCEPT" — you're looking for the wrong one
  • "ALWAYS" / "NEVER" — these are usually wrong; tech rarely works in absolutes

✂️ Elimination Technique

  1. Read the question stem carefully — what is it really asking?
  2. Cross out any answer that is obviously wrong or out of scope
  3. If 2 answers look similar, one of them is usually correct
  4. Look for answers that are "too extreme" — usually distractors
  5. If completely unsure, pick the more conservative/secure option

🖥️ Performance-Based Questions (PBQs)

PBQs appear at the start of the exam and are simulations (drag-drop, click-throughs, command line scenarios). Tips:

  • Don't panic — skip and come back if you're unsure
  • Read every instruction carefully before clicking
  • Most PBQs test basic knowledge in a visual way
  • Flagging and returning with fresh eyes often helps

📅 Day-Before Checklist

  • ✅ Review your weakest domains (check your tracker above!)
  • ✅ Look over mnemonics one more time
  • ✅ Get 8 hours of sleep — tired brains forget everything
  • ✅ Eat a good breakfast before the exam
  • ✅ Arrive early (or set up testing software ahead of time)
  • ✅ Bring valid photo ID if testing in-person

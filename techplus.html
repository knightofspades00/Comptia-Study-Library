Hardware Basics

Every computer system is built from a few core components that work together:

  • CPU (Central Processing Unit): The "brain" — executes instructions. Speed measured in GHz; modern CPUs have multiple cores (quad-core = 4 independent processors on one chip).
  • RAM (Random Access Memory): Fast, volatile workspace memory — cleared when power is off. More RAM = more programs running at once without slowdown.
  • Storage (HDD / SSD / NVMe): Non-volatile — persists data permanently. SSD (flash) is much faster than HDD (spinning disk). NVMe (via PCIe slot) is the fastest.
  • Motherboard: The main circuit board connecting all components — CPU, RAM, storage, GPU, and expansion slots.
  • GPU (Graphics Processing Unit): Dedicated processor for rendering images/video. Also used for AI and parallel computing.
💡 Volatile vs Non-Volatile: RAM is volatile (lost on power-off). Storage (SSD, HDD) is non-volatile (persists). This is a common exam question!
💼 Real-world

When you're shopping for a laptop, the three specs that decide how it actually feels to use are: CPU (how fast it thinks), RAM (how many things it can think about at once — 8 GB is the floor today, 16 GB is comfortable), and storage type (SSD beats HDD by a mile; an old laptop with an SSD upgrade often feels brand new).

Input & Output Devices

  • Input devices send data INTO the computer: keyboard, mouse, microphone, scanner, webcam, touchscreen.
  • Output devices present data FROM the computer to the user: monitor, printer, speakers, projector.
  • I/O devices do both: touchscreen (input + output), network interface card.

Number Systems

  • Binary (Base-2): Only 0s and 1s. All digital data is ultimately binary. Each position is a power of 2.
  • Bit: A single 0 or 1 — the smallest unit of digital data.
  • Byte: 8 bits. Can represent 256 values (2⁸). All storage is measured in bytes: KB (1,024 B), MB, GB, TB.
  • Hexadecimal (Base-16): Uses 0–9 and A–F. Commonly used for memory addresses and color codes.

🎯 Mnemonic — storage sizes

"Kids Make Good Teachers Proud" = KB → MB → GB → TB → PB. Each step is roughly ×1,000 (technically ×1,024, but rounded numbers are fine for exam ordering questions).

📌 Exam tip — bits vs bytes

An uppercase B = byte. A lowercase b = bit. Internet speed is in megabits per second (Mbps) — 100 Mbps = 12.5 MB/s, not 100. CompTIA tests whether you can spot the difference.

Networking Fundamentals

  • IP Address: A unique numerical label for every device on a network. IPv4 uses 32 bits (e.g., 192.168.1.10). IPv6 uses 128 bits for a much larger address space.
  • Router: Forwards data packets between different networks (e.g., your home network to the internet). Every packet passes through a router to leave its network.
  • Switch: Connects multiple devices within the same network (LAN). Sends data only to the intended destination device, unlike older hubs which broadcast to all.
  • Wi-Fi: IEEE 802.11 wireless networking. Common standards: 802.11ac (Wi-Fi 5) and 802.11ax (Wi-Fi 6). Uses radio waves; speed affected by distance and interference.
  • Internet vs. Intranet: The Internet is the global public network. An Intranet is a private internal network (like a company's internal website) accessible only from inside the organization.
📌 Exam tip — router vs switch

A switch connects devices inside one network; a router connects different networks. Your home Wi-Fi box is technically both — a router + switch + wireless access point in one. On the exam, "between networks" → router. "Within the same network" → switch.

Cloud Computing

  • Cloud Computing: Delivering computing services over the internet on a pay-as-you-go basis. Providers: AWS, Microsoft Azure, Google Cloud.
  • IaaS (Infrastructure as a Service): Rent virtual machines and storage. You manage the OS and apps. Example: AWS EC2.
  • PaaS (Platform as a Service): Rent an app development platform. Provider manages OS and runtime. Example: Heroku, Google App Engine.
  • SaaS (Software as a Service): Rent software accessed via browser. Provider manages everything. Example: Gmail, Microsoft 365, Salesforce.
ModelYou ManageProvider ManagesExample
IaaSOS, apps, dataHardware, network, virtualizationAWS EC2
PaaSApps, dataOS, runtime, middlewareHeroku
SaaSData, settings onlyEverything elseGmail, Office 365
⚠️ Exam Tip: "Which model gives you most control?" → IaaS. "Which requires least management?" → SaaS. The more you go up (IaaS → PaaS → SaaS), the more the provider manages.
🎯 Mnemonic — pizza analogy

Picture making pizza. IaaS = renting the kitchen (you bring everything else). PaaS = a pizza-making kit (dough & sauce included; you add toppings). SaaS = delivered pizza (you just eat it). The fancier the service, the less work for you — and the less control.

Operating Systems

The OS manages all hardware resources and provides a platform for applications to run. Key functions:

  • Manages CPU time (scheduling), RAM allocation, storage access, and I/O devices
  • Provides user interface (GUI or command line)
  • Handles security — user accounts, permissions, login

Common OS examples: Windows 11, macOS Ventura, Linux (Ubuntu, Fedora), Android, iOS

File Systems

The method an OS uses to organize and store files on storage media:

File SystemUsed ByNotes
NTFSWindowsSupports permissions, encryption, large files
APFSmacOSOptimized for SSDs, strong encryption
ext4LinuxReliable, journaling, widely used
FAT32UniversalWorks on all OS; max 4 GB file size
exFATUSB drivesCross-platform, no 4 GB limit
📌 Exam tip — the 4 GB FAT32 gotcha

You can't copy a single file larger than 4 GB onto a FAT32 USB drive — even if the drive is 64 GB total. This catches students every time. Format the drive as exFAT for large files that also need to work on both Mac and Windows.

Application Software vs. System Software

  • System software: Manages hardware and platform — OS, drivers, utilities (antivirus, backup tools).
  • Application software: Designed for end-user tasks — word processors, browsers, games, media players.

Programming Basics

  • Variable: A named storage location in code that holds a value. Example: age = 17
  • Algorithm: A step-by-step set of instructions to solve a problem. Every program is built from algorithms.
  • Conditional (If/Else): Makes decisions in code — "If X is true, do Y; otherwise do Z."
  • Loop: Repeats a block of code — for, while. Used to process lists of data.
  • Function: A reusable named block of code. Avoids repeating the same code in multiple places.
💡 Real-world example: When Netflix recommends a show, an algorithm analyzes your watch history, compares it to other users, and recommends shows with the highest predicted match.

Development Concepts

  • Source code: Human-readable instructions written in a programming language (Python, Java, JavaScript).
  • Compiler / Interpreter: Translates source code into machine instructions the CPU can execute.
  • Version control (Git): Tracks changes to code over time; allows teams to collaborate and roll back mistakes.
  • Bug: An error in code that causes unexpected behavior. Debugging = finding and fixing bugs.

What is a Database?

An organized collection of structured data managed by a DBMS (Database Management System). Databases power nearly every app — social media, banking, school records, online stores.

Relational Databases

  • Data is stored in tables (rows = records, columns = fields).
  • Tables are related to each other via common fields (foreign keys).
  • Examples: MySQL, PostgreSQL, Microsoft SQL Server, SQLite.

SQL Basics

SQL (Structured Query Language) is used to interact with relational databases:

CommandPurposeExample
SELECTRead / retrieve dataSELECT * FROM students
INSERTAdd new recordsINSERT INTO students VALUES (...)
UPDATEModify existing dataUPDATE students SET grade='A'
DELETERemove recordsDELETE FROM students WHERE id=5

NoSQL Databases

For unstructured or rapidly changing data. Types: document (MongoDB), key-value (Redis), graph, time-series. Used by social media, IoT, real-time analytics.

🔐 Security 20%

📺 Messer videos ▼

Common Threats

  • Malware: Malicious software designed to harm. Types: virus (attaches to files), worm (self-spreads), trojan (disguised), ransomware (encrypts files for ransom), spyware (steals data).
  • Phishing: Fake emails/messages designed to steal passwords or trick you into clicking malicious links. The #1 method attackers use to compromise accounts.
  • Social Engineering: Manipulating people (not computers) to give up information. Phishing is a type of social engineering.
📌 Exam tip — phishing variants

Same trick, different channel. Phishing = email. Smishing = SMS / text. Vishing = voice / phone. Whaling = phishing aimed at executives. If a question describes a fake text message, the answer is smishing, not "phishing."

Protection Methods

  • Firewall: Monitors and controls network traffic based on security rules. Can be hardware or software. Blocks unauthorized connections.
  • Strong Passwords: Long (12+ characters), complex (uppercase, lowercase, numbers, symbols), unique per account. Use a password manager.
  • Multi-Factor Authentication (MFA): Requires 2+ verification methods — something you know (password) + something you have (phone) + something you are (fingerprint).
  • Encryption: Scrambles data so only the intended recipient can read it. HTTPS uses encryption. Essential for protecting data in transit and at rest.
  • Software Updates / Patches: Fix security vulnerabilities. Unpatched software is one of the most common attack vectors.
⚠️ Real-World Rule: Never share your password — not even with "IT support." Legitimate IT staff never need your password. This is the #1 phishing social engineering script.
💼 Real-world — why MFA matters

Even a "leaked" password is mostly useless against an account with MFA, because the attacker still needs your second factor (phone, security key, biometric). Turn on MFA for your email first — your email is the master key to password resets for every other account.

🖥️ How a computer works — the IPOS cycle

IPOS is the model every computer follows: Input → Process → Output, with Storage feeding the cycle. The CPU's job (inside PROCESS) is its own tiny loop — fetch, decode, execute, store — billions of times per second. When you "open a program," the OS loads it from Storage into RAM, then the CPU runs the fetch-execute cycle until you close it. Pull the plug — RAM clears, but Storage keeps your work.

📏 Storage size scale — from a bit to a petabyte

Bars are scaled visually so the jump from bit to PB reads at a glance — but remember each real step is ~1,000× bigger than the one before. Internet speed uses lowercase b (bits/s); file size uses uppercase B (bytes). 100 Mbps internet ≈ 12 MB/s download.

☁️ Cloud service models — who manages what

You always own your data, even on SaaS — a CompTIA exam favorite. As you move right (IaaS → SaaS) you give up control but gain convenience.

🌐 Local network vs the internet

A switch talks within your network. A router talks between networks. A modem translates between your router and the ISP. Most home Wi-Fi boxes combine all three.

🔐 HTTPS — what encryption actually does

HTTP and HTTPS send the same data — only HTTPS scrambles it on the wire. The padlock 🔒 in the address bar means TLS is on and the certificate matched the site. That's why public Wi-Fi is safe for HTTPS but risky for plain HTTP.

🛡️ Malware family tree (simplified)

The key CompTIA distinction: a virus needs a host file and a user to spread; a worm spreads itself across networks. Treat any installed software as a trojan suspect if you didn't get it from a verified source.

🗄️ Database vs file storage

Databases use SQL to ask precise questions of structured data: "give me every customer in Boston" is one line of SQL but a tedious manual search across a folder of files.

🧠 Walk Me Through It — Step-by-Step Workings

The three concepts students most often get stuck on. Click any panel to expand the worked example.

🔢How do I convert decimal ↔ binary?
Scenario: Convert 130 to binary, then turn 10110101 back into decimal.
Step 1 — Memorize the powers of 2

Each bit position holds a power of 2. For an 8-bit byte:

Bit pos76543210
Value1286432168421
Step 2 — Decimal → Binary: 130

Walk left to right. Subtract the biggest power of 2 you can; write a 1. If you can't, write 0.

  • 130 − 128 = 2 → bit 7 = 1
  • 2 < 64, 32, 16, 8, 4 → bits 6, 5, 4, 3, 2 = 0
  • 2 − 2 = 0 → bit 1 = 1
  • bit 0 = 0

Result: 10000010

Step 3 — Binary → Decimal: 10110101

Add the values of every position that has a 1.

  • bit 7 (128) + bit 5 (32) + bit 4 (16) + bit 2 (4) + bit 0 (1)
  • 128 + 32 + 16 + 4 + 1 = 181
Answers

130 ⇒ 10000010 · 10110101 ⇒ 181

This same routine works for every octet of every IPv4 address — practice until it's automatic.

🌐Why does /24 mean 256 addresses?
Scenario: Your teacher writes 192.168.1.0/24 on the board. What does the /24 mean and how big is that network?
Step 1 — IPs are 32 bits

An IPv4 address is just 32 bits, written as 4 octets of 8 bits each. The slash tells you how many bits from the left are fixed as the network part.

Step 2 — /24 means "first 24 bits are network"

That leaves 32 − 24 = 8 bits free to identify hosts inside the network.

Step 3 — Count the host combinations

Each free bit doubles the count. 8 free bits → 2⁸ = 256 total addresses (numbered .0 through .255).

Two of those are reserved: the network address (.0, all host bits 0) and the broadcast address (.255, all host bits 1). So 254 usable hosts.

Takeaway

The general formula: usable hosts = 2^(32 − prefix) − 2. Smaller prefix = bigger network. Bigger prefix = smaller network. /24 is a typical home or small-office LAN size.

🔐What do file permissions rwxr-x--- actually mean?
Scenario: You see -rwxr-x--- next to a file on a Linux box. Who can do what?
Step 1 — Break it into 4 parts
CharsPositionMeaning
-File type- file · d directory · l symlink
rwxOwnerThe user who owns the file
r-xGroupMembers of the file's group
---OtherEveryone else
Step 2 — Decode each set of 3

r = read, w = write, x = execute. A dash means that permission is denied.

  • Owner has rwx → can read, write, run it
  • Group has r-x → can read and run, NOT modify
  • Other has --- → no access at all
Step 3 — Octal shortcut

Each set of 3 maps to an octal digit: r=4, w=2, x=1. Add them up.

  • rwx = 4+2+1 = 7
  • r-x = 4+0+1 = 5
  • --- = 0+0+0 = 0

So rwxr-x--- = chmod 750.

Cheat sheet
  • chmod 777 = everyone gets everything (rarely correct — dangerous)
  • chmod 755 = owner full, everyone read+execute (typical script)
  • chmod 700 = private to the owner only (SSH keys, secrets)
  • chmod 644 = owner can write, everyone else can read (config files)

🧠 Memory Tricks & Mnemonics
🖥️ Hardware vs Software — The Golden Rule
"If you can kick it, it's hardware. If you can only curse at it, it's software."
Hardware = physical components you can touch (CPU, RAM, keyboard, monitor) · Software = programs and instructions (Windows, Chrome, Microsoft Word) · Firmware = software permanently stored in hardware chips (BIOS)
🧠 Computer Memory Hierarchy — "CRSD"
"Computers Remember Stuff Differently"
⚡ Cache — fastest, smallest, inside the CPU 🔵 RAM — fast, temporary, loses data when off 💾 Storage (SSD/HDD) — slower, permanent, large ☁️ Drive/Cloud — largest, slowest, off-device
📥 Input vs Output — "I-O, I-O, it's off to work we go"
"INPUT goes INTO the computer · OUTPUT comes OUT of the computer"
⌨️ Input: keyboard, mouse, microphone, camera, scanner 🖥️ Output: monitor, printer, speakers, projector ↔️ Both: touchscreen, USB drive (reads AND writes)
🌐 Network Devices — "The Highway Crew"
"Hub = dump truck (broadcasts), Switch = smart sorter (direct delivery), Router = GPS navigator (between networks)"
Hub: sends data to everyone on the network (old, inefficient) · Switch: sends data only to the intended recipient (smart) · Router: connects different networks together (e.g., your home to the internet) · Modem: converts internet signal from your ISP into usable data
🔒 Security Basics — "Lock the 3 Doors"
"Something you KNOW (password), something you HAVE (phone), something you ARE (fingerprint)"
These are the 3 authentication factors. Using 2 or more = Multi-Factor Authentication (MFA) — much more secure than just a password! Strong password = 12+ characters, mix of letters/numbers/symbols, no dictionary words.
☁️ Cloud Service Models — "Pizza as a Service"
"IaaS = ingredients delivered · PaaS = dough made for you · SaaS = pizza delivered ready to eat"
IaaS = you manage OS, apps (AWS EC2) PaaS = platform ready, you write code (Heroku) SaaS = fully managed app (Gmail, Salesforce)

🎯 How to Pass the CompTIA Tech+ (FC0-U71)

🗣️ CompTIA Question Language

CompTIA uses specific trigger words that change the right answer. Learn to spot them:

  • "MOST likely" — eliminate impossible answers; pick the best fit
  • "BEST" — there may be 2 correct answers; one is more efficient/secure
  • "FIRST" — follow the correct order/process (e.g., troubleshooting steps)
  • "LEAST" / "EXCEPT" — you're looking for the wrong one
  • "ALWAYS" / "NEVER" — these are usually wrong; tech rarely works in absolutes

✂️ Elimination Technique

  1. Read the question stem carefully — what is it really asking?
  2. Cross out any answer that is obviously wrong or out of scope
  3. If 2 answers look similar, one of them is usually correct
  4. Look for answers that are "too extreme" — usually distractors
  5. If completely unsure, pick the more conservative/secure option

🖥️ Performance-Based Questions (PBQs)

PBQs appear at the start of the exam and are simulations (drag-drop, click-throughs, command line scenarios). Tips:

  • Don't panic — skip and come back if you're unsure
  • Read every instruction carefully before clicking
  • Most PBQs test basic knowledge in a visual way
  • Flagging and returning with fresh eyes often helps

📅 Day-Before Checklist

  • ✅ Review your weakest domains (check your tracker above!)
  • ✅ Look over mnemonics one more time

🌉 Ready for the next level?

From Tech+ → A+

Tech+ covers what the technology is. A+ covers how to support it — installing it, configuring it, fixing it when it breaks. If Tech+ felt comfortable, you're already ~30% of the way to A+.

What carries over

  • Hardware fundamentals (CPU, RAM, storage)
  • Network basics (IP, router, Wi-Fi)
  • OS concepts (Windows, macOS, Linux)
  • Security basics (malware, phishing, passwords)
  • Binary, file systems, cloud models

What's NEW in A+

  • Two exams: Core 1 (220-1101) + Core 2 (220-1102) — both required
  • Performance-based questions (PBQs) — drag-and-drop, simulations
  • Mobile device hardware & troubleshooting (laptops, phones)
  • Windows command line: sfc, chkdsk, diskpart, net use
  • Printers, cabling, video cards in depth
  • Active Directory, GPOs, account management
  • CompTIA's 6-step troubleshooting methodology
  • Professional behavior & soft skills (customer service)

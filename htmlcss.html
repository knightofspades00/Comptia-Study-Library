🏗️ 1.0 — HTML Foundations & Document Structure 12%

📚 W3Schools ▼

The anatomy of an HTML document

Every web page follows the same skeleton. Memorize it — most exam questions assume you can sketch this from scratch.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page title shown in the tab</title>
</head>
<body>
  <!-- visible content lives here -->
</body>
</html>
  • <!DOCTYPE html> — tells the browser this is HTML5. Must be the first line. Not technically a tag.
  • <html> — the root element. The lang attribute helps screen readers and search engines.
  • <head> — metadata, links to CSS/JS, page title, viewport settings. Not displayed.
  • <body> — everything the user sees on the page.
  • <meta charset="UTF-8"> — character encoding. Always UTF-8 in 2025.
  • <meta name="viewport"> — controls mobile scaling. Without it, phones render at a desktop width and shrink.

Tags, elements, attributes — terminology

  • Tag — the literal text between angle brackets: <p> is the opening tag, </p> the closing.
  • Element — the opening tag, content, and closing tag together. A complete unit.
  • Attribute — key/value pair inside the opening tag. <a href="..." target="_blank"> has two attributes.
  • Void / self-closing — elements with no content or closing tag: <img>, <br>, <hr>, <input>, <meta>, <link>.

Headings, paragraphs, comments

  • Headings: <h1> through <h6>. Only one <h1> per page (best practice). Don't skip levels — h1 → h2 → h3, not h1 → h4.
  • Paragraphs: <p> wraps body text. Multiple spaces and line breaks collapse in HTML — use <br> for an intentional line break.
  • Comments: <!-- like this -->. Visible in View Source, so never put secrets in HTML comments.
  • Horizontal rule: <hr> — thematic break, void element.

Block vs inline (the foundational distinction)

Default displayBehaviorCommon elements
BlockTakes full width, forces a new line before and after, can have width/height/margin/padding all four sidesdiv, p, h1–h6, ul, ol, li, section, article, nav, header, footer
InlineOnly as wide as content, sits next to siblings on the same line, can't set width/height directlyspan, a, strong, em, img, code, label, input
📌 Exam favorite: "Why doesn't my <span> respect width: 200px?" — inline elements ignore explicit width and height. Switch its display to inline-block or block via CSS.

The class and id attributes

  • id — unique per page. Used as an anchor target (#section1) and as a single-target CSS/JS hook.
  • class — reusable. Multiple elements can share a class. An element can have many classes: class="btn btn-primary large".
  • data-* — custom attributes for JS hooks: data-user-id="42". Accessed via element.dataset.userId.

<strong>Strong importance — typically boldThe word is semantically critical, not just visually heavy
<em>Emphasis — typically italicThe word would be stressed when read aloud
<b>Visually bold, no extra meaningLike a keyword in a glossary — bold without importance
<i>Visually italic, no extra meaningForeign phrases, ship names, taxonomic names

<pre>Preserves whitespaceMulti-line code blocks, ASCII art
<blockquote>Block-level quotationLong quote (usually indented by default)

Lists

  • Unordered: <ul> with <li> children. Renders as bullets by default.
  • Ordered: <ol> with <li> children. Renders as numbers. Use type="A", "a", "I", "i" for letters/Roman numerals. start="5" begins at 5.
  • Description list: <dl> wraps <dt> (term) and <dd> (description) pairs. Great for glossaries and FAQs.
  • Lists can nest — put a <ul> inside a <li>.

Links

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Open example</a>
  • href — destination. Absolute (https://...), relative (../about.html), anchor (#section1), or special (mailto:, tel:).
  • target="_blank" — opens in a new tab. Always pair with rel="noopener noreferrer" for security (prevents the new page from accessing window.opener).
  • title — tooltip on hover. Optional.
  • download — turns the link into a file download.

Images

<img src="cat.jpg" alt="A grey tabby cat asleep on a keyboard" width="400">
  • src — image URL.
  • alt — descriptive text for screen readers and broken-image fallback. Always include it, even if empty (alt="") for decorative images. Exam favorite.
  • width and height — set as attributes (no units) to reserve space and avoid layout shift. Override with CSS as needed.
  • loading="lazy" — defers off-screen images until the user scrolls near them.
  • <picture> with <source> — responsive images per breakpoint or format (WebP fallback to JPG).

Multimedia

  • <audio> — embed audio. controls shows the player. Multiple <source> children let the browser pick a supported format.
  • <video> — embed video. controls, autoplay, muted, loop, poster. Autoplay only works if muted (browser policy).
  • <iframe> — embed another page (YouTube videos, maps). Use sandbox and loading="lazy" in production.
💼 Real-world — broken images and SEO

Search engines read alt text to understand what an image shows. A page full of alt="" or missing alt attributes ranks worse — and is unusable for blind students. This is also why alt is the #1 most-tested image attribute.

📋 3.0 — Tables, Forms & Input 14%

📚 W3Schools ▼

Tables — for tabular data, not for layout

<table>
  <thead>
    <tr><th>Name</th><th>Score</th></tr>
  </thead>
  <tbody>
    <tr><td>Alice</td><td>92</td></tr>
    <tr><td>Bob</td><td>88</td></tr>
  </tbody>
</table>
  • <table> wraps the whole thing. <tr> = table row. <th> = header cell. <td> = data cell.
  • <thead>, <tbody>, <tfoot> — semantic groupings. Useful for styling and screen readers.
  • colspan / rowspan — make a cell span multiple columns or rows.
  • <caption> — table title. Must be the first child of <table>.
  • Don't use tables for page layout — use CSS Flexbox or Grid. Tables are for actual rows-and-columns data.

Form basics

<form action="/submit" method="post">
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">Send</button>
</form>
  • action — URL the form submits to. Omit for current page.
  • method — GET (data in URL, idempotent, public) or POST (data in body, used for changes).
  • name — the key that gets sent on submit. email=alice@example.com. If an input has no name, it is not submitted.
  • id — used by the <label for="..."> to link the label to the input. Tapping the label focuses the input.

Input types — the high-yield list

typeWhat it does
textSingle-line text (default)
passwordMasked text
emailText + built-in @-format validation, mobile shows @-key
telPhone — mobile shows number pad
urlURL with built-in validation
numberNumeric, with min / max / step
rangeSlider control
date / time / datetime-localDate / time pickers
checkboxTrue/false toggle. Many can be checked at once.
radioOne-of-many group (share the same name)
fileFile upload — use accept="image/*" to filter
colorColor picker
hiddenNot visible — used for tokens, IDs sent on submit
submit / reset / buttonForm action buttons

Other form controls

  • <textarea> — multi-line text. Use rows + cols or size via CSS.
  • <select> with <option> children — dropdown. Add multiple to allow multiple selections.
  • <optgroup label="..."> — group options inside a select.
  • <fieldset> with <legend> — group related inputs with a visual border + label.
  • <datalist> — autocomplete suggestions for an <input> via list="...".

Validation attributes

  • required — must have a value.
  • min / max / step — numeric or date ranges.
  • minlength / maxlength — string lengths.
  • pattern — regex the value must match.
  • placeholder — gray hint text, disappears on focus. Not a label replacement — exam trap.
📌 Exam tip — name vs id

name is what the server sees on submit. id is what the browser uses to link a label to an input and what JavaScript uses to find the element. They often share a value, but they serve different purposes.

Semantic elements — describe what content is

Before HTML5, every container was a <div>. HTML5 added semantic tags that tell the browser, screen readers, and search engines what a region of the page means.

ElementWhat it represents
<header>Introductory content / nav. Can appear inside <article> too.
<nav>Major navigation links
<main>The primary content of the page. Only one per document.
<section>A thematic grouping of content — usually has a heading
<article>Self-contained, distributable content (blog post, news story, comment)
<aside>Tangentially related content — sidebars, pull quotes
<footer>Closing content — copyright, contact, related links
<figure> + <figcaption>An illustration plus caption — image, diagram, code listing
<time datetime="2026-05-13">Machine-readable date / time
<details> + <summary>Native expandable panel — no JS needed (same one this hub uses)
<dialog>Native modal dialog (HTML5)

Canvas vs SVG

Feature<canvas><svg>
RenderingPixel-based (bitmap)Vector-based (scalable shapes)
ResolutionFixed — pixelates on zoomCrisp at any zoom level
DOM accessThe canvas is one element; shapes inside aren't DOM nodesEvery shape IS a DOM node — selectable, styleable, scriptable
Best forGames, video processing, photo editing, particle effectsIcons, charts, diagrams, anything that needs to scale or be styled

HTML5 storage APIs

  • localStorage — key/value pairs, persists forever (until cleared). ~5–10 MB per origin. This is what every CompTIA hub in this library uses for progress tracking.
  • sessionStorage — same API as localStorage but cleared when the tab closes.
  • cookies — older mechanism; sent with every HTTP request automatically. ~4 KB limit. Mostly for server-side authentication.
  • IndexedDB — full client-side database for large structured data (offline-first apps).
// Save
localStorage.setItem('username', 'alice');
// Read
const name = localStorage.getItem('username');
// Remove
localStorage.removeItem('username');

Other web APIs worth knowing

  • Geolocation: navigator.geolocation.getCurrentPosition(callback) — asks the user for permission, returns lat/long.
  • Drag & Drop: any element with draggable="true". Listen for dragstart, dragover, drop.
  • Fetch API: modern replacement for XMLHttpRequest. Returns Promises. fetch(url).then(r => r.json()).
  • Web Workers: background JS threads for CPU-heavy work without freezing the UI.
  • Server-Sent Events (SSE): server-to-client one-way push over HTTP.
  • WebSockets: full-duplex client/server messaging — chat apps, live dashboards.

Accessibility — non-negotiable for the exam and the real world

  • Semantic tags are accessibility for free — <nav> tells a screen reader "this is the navigation".
  • alt text on images — every image without exception.
  • Labels for form inputs — every input needs an associated <label> (via for="" or by wrapping the input).
  • Logical heading hierarchy — don't skip levels; screen reader users navigate by headings.
  • Color isn't the only signal — don't communicate via color alone (red = error needs an icon and text too).
  • ARIA attributes — use sparingly; semantic HTML first. aria-label, aria-hidden, role="" when nothing native fits.

🎨 5.0 — CSS Fundamentals & Selectors 15%

📚 W3Schools ▼

Three ways to apply CSS

  • Inline — style="" attribute on an element. Highest specificity, but mixes content and presentation. Avoid.
  • Internal — <style> block in <head>. Useful for one-off pages or quick demos.
  • External — separate .css file linked from the head: <link rel="stylesheet" href="styles.css">. The standard for any real project — one stylesheet styles every page.

Selector basics

SelectorTargets
*Universal — every element
pAll <p> elements
.warningAnything with class="warning"
#headerThe element with id="header"
p.warning<p> elements that also have class="warning"
h1, h2, h3Group selector — all h1, h2, h3
nav aDescendant — any <a> inside a <nav> (at any depth)
nav > aDirect child — only <a> that's a direct child of <nav>
h2 + pAdjacent sibling — the <p> immediately after an <h2>
h2 ~ pGeneral sibling — every <p> after an <h2> at the same level
[type="email"]Attribute selector — inputs with type=email

Pseudo-classes and pseudo-elements

  • Pseudo-classes (single colon) — describe a state. :hover, :focus, :active, :visited, :checked, :disabled, :first-child, :nth-child(odd), :not(.skip).
  • Pseudo-elements (double colon) — describe a part of the element. ::before, ::after (insert generated content), ::first-line, ::first-letter, ::placeholder, ::selection.

Specificity — when two rules fight, the winner is...

CSS resolves conflicts by scoring each selector and the higher score wins. Score is read like a 4-digit number: (inline, IDs, classes/attrs/pseudo-classes, elements/pseudo-elements).

SelectorScore
p0,0,0,1
.title0,0,1,0
nav a:hover0,0,1,2
#header0,1,0,0
style="..." (inline)1,0,0,0
!importantnuke — overrides everything above. Use sparingly.
📌 Specificity tie-breaker: If two selectors have the same score, whichever appears LATER in the CSS wins. This is the C in "Cascade" — cascading style sheets cascade top to bottom.

Inheritance

Some properties (mostly text-related — color, font-family, font-size, line-height, text-align) inherit from parent to child by default. Most others (border, padding, margin, background) do not. You can force inheritance with property: inherit; on any property.

Common value units

  • px — absolute pixels. Reliable, but doesn't scale with user font preferences.
  • % — percentage of the parent element's same dimension.
  • em — relative to the current element's font-size. Compounds in nested elements.
  • rem — relative to the root (<html>) font-size. Doesn't compound — most predictable for layout.
  • vw / vh — 1% of viewport width / height. 100vh = full viewport height.
  • auto — let the browser decide. Magic for margins and grid.
  • fr — fractional unit (Grid only) — distributes remaining space.

📦 6.0 — Box Model, Display & Position 15%

📚 W3Schools ▼

The box model — every element is a box

Each element on the page is a rectangle composed of four concentric layers, from the inside out:

  1. Content — the actual text or child elements. Has a width and height.
  2. Padding — inner spacing between the content and the border. Background color extends into padding.
  3. Border — the line drawn around the padding.
  4. Margin — outer spacing between this element and its neighbors. Transparent.

By default (box-sizing: content-box), the width you set applies only to the content area. Border and padding are added on top, making the actual rendered width larger than you specified.

/* Best practice: reset everything to border-box */
*, *::before, *::after { box-sizing: border-box; }

/* Now width includes content + padding + border */
.card { width: 300px; padding: 20px; border: 2px solid; }
/* Renders at exactly 300px wide */

Margin shorthand

  • margin: 10px; — all four sides
  • margin: 10px 20px; — top/bottom 10, left/right 20
  • margin: 10px 20px 30px; — top 10, left/right 20, bottom 30
  • margin: 10px 20px 30px 40px; — top, right, bottom, left (clockwise from top)
  • margin: 0 auto; — top/bottom 0, left/right auto = horizontally center a block element with an explicit width

Padding uses the same shorthand syntax.

Margin collapse — the classic gotcha

When two block elements stack and both have a vertical margin, the larger margin "wins" — they don't add up. 20px bottom margin on element A + 15px top margin on element B = 20px total gap, not 35px. Only happens vertically, only between block elements, only on margins (not padding).

The display property — controls how an element flows

displayBehavior
blockFull width, new line before/after, accepts width/height
inlineWidth = content, sits in a line, ignores width/height
inline-blockSits in a line but accepts width/height — useful for buttons and chips
flexTurns this element into a flex container; children become flex items
gridTurns this element into a grid container
noneElement is removed entirely from layout (vs visibility: hidden which hides but reserves space)

Position — taking elements out of normal flow

positionBehavior
static (default)Normal document flow. top/right/bottom/left have no effect.
relativeStill in flow, but top/right/bottom/left offset it from where it would have rendered. Also becomes a positioning context for absolute children.
absoluteRemoved from flow. Positions relative to the nearest non-static ancestor (or the viewport if none).
fixedRemoved from flow. Positions relative to the viewport — stays in place when you scroll. Sticky navbars.
stickyActs like relative until it hits a scroll threshold, then like fixed. Section headers that pin at the top.

z-index only works on elements with a non-static position. Higher numbers render on top. Default is auto.

Overflow

  • overflow: visible; (default) — content spills out
  • overflow: hidden; — clipped, no scrollbar
  • overflow: scroll; — always shows scrollbars
  • overflow: auto; — shows scrollbars only when needed
  • overflow-x and overflow-y set horizontal/vertical separately
💼 Real-world — why everyone resets to border-box

The default content-box sizing breaks intuition: "I asked for 300px wide, why is it 344px?" After enough projects, every team adds * { box-sizing: border-box; } as the first rule. Tailwind, Bootstrap, and most modern frameworks do this automatically.

📐 7.0 — Flexbox & Grid Layout 14%

📚 W3Schools ▼

Flexbox — 1-dimensional layout (rows OR columns)

Flex distributes items along one axis. Set display: flex on the container and its direct children become flex items.

Container properties

PropertyWhat it doesCommon values
flex-directionMain axis directionrow (default), column, row-reverse
justify-contentAlignment along the main axisflex-start, center, flex-end, space-between, space-around, space-evenly
align-itemsAlignment along the cross axisstretch (default), center, flex-start, flex-end, baseline
flex-wrapWrap to next line when items overflownowrap (default), wrap
gapSpace between items (replaces margin tricks)16px, 1rem 2rem

Item properties

  • flex-grow — relative share of extra space. Default 0.
  • flex-shrink — relative share of shrinkage when space is tight. Default 1.
  • flex-basis — starting size before grow/shrink. Default auto.
  • flex: 1; — shorthand for 1 1 0. The classic "fill remaining space evenly."
  • align-self — override align-items for just this item.
  • order — change visual order without changing source order.
/* The classic centered card */
.parent { display: flex; justify-content: center; align-items: center; min-height: 100vh; }

/* The classic nav: logo left, links right */
.nav { display: flex; justify-content: space-between; align-items: center; }

/* Three equal columns */
.row { display: flex; gap: 20px; }
.col { flex: 1; }

CSS Grid — 2-dimensional layout (rows AND columns)

Use Grid when you have a true grid — rows and columns that align. Flex handles one axis at a time; Grid handles both.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto;
  gap: 20px;
}

Grid track sizing

  • fr — fractional unit. 1fr 2fr 1fr = three columns, middle is twice as wide.
  • repeat(N, size) — repeat a track pattern. repeat(12, 1fr) = 12 equal columns (classic Bootstrap-style grid).
  • minmax(min, max) — track sizes between a minimum and maximum. minmax(200px, 1fr) = at least 200px, share extra space.
  • auto-fit / auto-fill — let the browser decide how many columns fit. repeat(auto-fit, minmax(250px, 1fr)) is the magic responsive grid line — fits as many 250px+ columns as possible.

Placing items in the grid

  • grid-column: 1 / 3; — start at line 1, end at line 3 (spans 2 columns).
  • grid-column: span 2; — span 2 columns, auto-placed.
  • grid-row: 1 / -1; — span the full height of the grid.
  • grid-template-areas — name regions and place items by name. Best for page layouts.
.layout {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}
.layout header  { grid-area: header; }
.layout aside   { grid-area: sidebar; }
.layout main    { grid-area: main; }
.layout footer  { grid-area: footer; }
📌 Flex vs Grid — when to pick which

If your layout is a row or column (nav bar, card list, button group) → Flex. If it has rows AND columns that align (full-page layout, gallery, dashboard) → Grid. They can nest — Grid for the page shell, Flex for components inside each region.

📱 8.0 — Responsive Design, Transitions & Animation 6%

📚 W3Schools ▼

Responsive web design — same HTML, different layouts

The viewport meta tag <meta name="viewport" content="width=device-width, initial-scale=1.0"> is the prerequisite. Without it, mobile devices render at a "fake" 980px width and shrink the page.

Media queries

/* Mobile first — base styles apply everywhere */
.container { padding: 16px; }

/* Tablet and up */
@media (min-width: 768px) {
  .container { padding: 32px; max-width: 720px; margin: 0 auto; }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .container { max-width: 960px; }
}

Common breakpoints (not standardized, but widely used)

RangeCommon breakpointDevice
< 640px—Phone (portrait)
640–768pxmin-width: 640pxPhone (landscape), small tablet
768–1024pxmin-width: 768pxTablet
1024–1280pxmin-width: 1024pxLaptop
1280px+min-width: 1280pxDesktop monitor

Other media query features

  • @media (orientation: landscape) — orientation
  • @media (prefers-color-scheme: dark) — user's OS theme
  • @media (prefers-reduced-motion: reduce) — accessibility: user wants less animation
  • @media (hover: hover) — true on devices with a mouse, false on touch
  • @media print — styles for print preview / printed page

Mobile-first vs desktop-first

Mobile-first: base CSS targets the smallest screen. Media queries add styling for larger screens using min-width. Desktop-first: base CSS targets desktop, media queries strip / simplify for smaller screens using max-width. Mobile-first is the modern default — phone traffic is the majority and the base rules are simpler.

Transitions — animate between two states

.btn {
  background: #3498db;
  transform: scale(1);
  transition: background 0.2s ease, transform 0.15s ease;
}
.btn:hover {
  background: #2980b9;
  transform: scale(1.05);
}
  • transition-property — which property to animate (all, or specific names).
  • transition-duration — how long (e.g. 0.3s).
  • transition-timing-function — easing curve. ease (default), linear, ease-in, ease-out, ease-in-out, cubic-bezier(...).
  • transition-delay — wait before starting.

Transforms

  • translate(x, y) — move without affecting layout. translateX(100px), translateY(-50%).
  • rotate(deg) — spin in 2D. rotate(45deg).
  • scale(n) — resize. scale(1.2) = 120%.
  • skew(deg) — slant.
  • Combine in one declaration: transform: translateX(50px) rotate(15deg) scale(1.1);

Keyframe animations — full-fledged motion

@keyframes pulse {
  0%   { transform: scale(1); opacity: 1; }
  50%  { transform: scale(1.1); opacity: 0.7; }
  100% { transform: scale(1); opacity: 1; }
}

.notification {
  animation: pulse 1.5s ease-in-out infinite;
}
  • @keyframes name { ... } defines the steps.
  • animation shorthand: name, duration, timing-function, delay, iteration-count, direction, fill-mode.
  • animation-iteration-count: infinite loops forever.
  • animation-fill-mode: forwards retains the final state after animation ends.
📌 Performance tip

Animate transform and opacity for smooth 60fps motion — the browser GPU-accelerates these. Animating width, height, top, left triggers layout recalculation every frame and stutters on phones. Exam favorite.

Activity 1: Build a personal homepage

The single best HTML/CSS exercise: build a one-page personal site about yourself. It uses every fundamental — heading hierarchy, paragraphs, an image, links, a list, a contact form, basic CSS for fonts/colors/layout. The hard part is restraint, not technique.

📦 The CSS Box Model

Every element is a stack of four concentric boxes. Master this and most layout bugs become obvious.

Modern best practice
Reset every element to box-sizing: border-box. Now width: 300px means exactly 300px wide — content shrinks to make room for padding and border. Every modern CSS framework (Tailwind, Bootstrap, etc.) ships with this reset.

📐 Flexbox — One Axis at a Time

Set display: flex on a container. Children become flex items arranged along the main axis. The cross axis is perpendicular.

Flex one-liners worth memorizing
Center anything horizontally + vertically: on the parent — display: flex; justify-content: center; align-items: center;. Equal-width columns: display: flex; gap: 20px; on parent, flex: 1; on each child.

🎯 CSS Specificity — Who Wins?

When two rules target the same element with conflicting properties, the browser picks the one with higher specificity. The score reads like a four-digit number from left to right.

🧠 Walk Me Through It — Step-by-Step Workings

The three things students most often get stuck on. Click any panel to expand the worked example.

📦The box model — why is my "300px wide" box actually 344px?
Scenario: You write .card { width: 300px; padding: 20px; border: 2px solid; }. The page renders the card at 344px wide. Where does the extra 44px come from?
Step 1 — Default box-sizing is content-box

By default, the width property sets only the content area. Padding and border are added on top.

  • Content: 300px
  • + left padding 20px + right padding 20px = +40px
  • + left border 2px + right border 2px = +4px
  • Total rendered width = 344px
Step 2 — Fix: switch to border-box

With box-sizing: border-box, the width includes padding and border. Content shrinks to make room.

*, *::before, *::after {
  box-sizing: border-box;
}

Now width: 300px means exactly 300px wide. Content is 256px after subtracting padding (40px) and border (4px).

Step 3 — What about margin?

Margin always sits OUTSIDE the box. Even in border-box mode, margin adds extra space around the element — it does not count toward the width. Two adjacent vertical margins also collapse: 30px bottom on element A + 20px top on element B = 30px total, not 50px. (Padding does not collapse.)

Exam takeaway

Default: content-box — width counts only content. Modern reset: border-box — width counts content + padding + border. Always start a project with the reset. Margin never counts toward width and can collapse vertically.

🎯Specificity face-off — why won\'t my .card { color: red } apply?
Scenario: Your CSS has #sidebar p { color: blue; } .card { color: red; }. A paragraph in #sidebar with class card renders BLUE. You expected red. What happened?
Step 1 — Compute each selector\'s specificity
SelectorScore (inline, ID, class, element)
#sidebar p0, 1, 0, 1 — one ID + one element
.card0, 0, 1, 0 — one class
Step 2 — Compare left to right, digit by digit

Read the score like a four-digit number. First non-tie wins.

  • Position 1 (inline): 0 vs 0 — tie
  • Position 2 (IDs): 1 vs 0 — #sidebar p wins, blue applies
Step 3 — How to fix it (three options, ordered by cleanliness)
  1. Add specificity to .card: #sidebar p.card { color: red; } — now 0,1,1,1 vs 0,1,0,1 — red wins.
  2. Drop the ID: rewrite #sidebar p as .sidebar p — score becomes 0,0,1,1, and .card alone (0,0,1,0) still loses on element count. So combine: .sidebar .card = 0,0,2,0 — wins.
  3. Use !important: .card { color: red !important; } — works, but escalates. The next override needs its own !important too. Avoid in a clean codebase.
Exam decision rule

Avoid IDs in CSS selectors when possible — they win every class-based fight and lock you into !important later. Use classes for everything stylistic; reserve IDs for anchors and JS hooks.

🔧Flexbox vs Grid — which one do I pick?
Scenario: You\'re building a page with a logo + nav at the top, three equal feature cards below, then a footer. Which CSS layout system fits each region?
Step 1 — Ask: 1D or 2D?
  • One row OR one column → Flexbox
  • Rows AND columns that align across both → Grid

It\'s OK — and very common — to nest them: Grid for the page shell, Flex for components inside each region.

Step 2 — Apply to the scenario, region by region
RegionShapePick
Top: logo left + nav rightOne row, two endsFlexbox — display:flex; justify-content:space-between;
Three feature cardsOne row, equal widthsFlexbox — display:flex; gap:20px; + flex:1 on each card
OR three cards that wrap to two then one column on smaller screensResponsive 2DGrid — grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
FooterOne rowFlexbox usually fine
Step 3 — The "auto-fit minmax" superpower

For the cards row, repeat(auto-fit, minmax(250px, 1fr)) is a one-liner responsive layout. The grid fits as many ≥250px columns as the viewport allows; remaining space distributes equally. No media queries needed for the basic case.

When in doubt

Start with Flexbox. Reach for Grid when you have a true 2D structure (gallery, dashboard, full-page layout with named regions). They work great nested — most modern sites use Grid for the page shell + Flex inside each section.

🧠 Tips, Habits & Careers

Stuff that\'s not on the exam but matters for the real job.

🎯 Habits that make HTML/CSS click

  • Always start a project with a CSS reset. * { box-sizing: border-box; margin: 0; padding: 0; }. Saves hours of "why is there a gap" debugging.
  • Mobile-first. Write CSS for the smallest screen, then add @media (min-width: ...) for larger. Cleaner code, better default, faster mobile.
  • Use the browser DevTools every day. Right-click → Inspect. Hovering elements highlights them. Toggling CSS rules teaches you what each property does, fast.
  • Read the cascade out loud. When a style doesn\'t apply, the bug is usually specificity or source order — articulate which rule is winning and why.
  • Semantic HTML first, ARIA last. If the right element exists (<button>, <nav>, <label>), use it. ARIA is the duct tape when no semantic option exists.
  • Don\'t style with !important. Once you do, the next override needs !important too, and you\'re in escalation hell. Fix specificity instead.

💼 Where this leads (entry-level roles)

Front-End Developer (junior)

Builds the UI of web apps. Day-to-day: HTML, CSS, JavaScript, a framework like React or Vue. The W3Schools certs prove you have the foundation; portfolio projects prove you can ship.

Web Designer / Producer

Turns design mocks (Figma, Sketch) into actual web pages. Bridges design and engineering. Strong CSS chops are the entire skill.

Email Developer

Builds HTML email templates that render in Gmail, Outlook, Apple Mail — each with quirks. Niche but well-paid because few developers know it.

WordPress / CMS Theme Developer

Customizes themes for WordPress, Shopify, Webflow. Often freelance — HTML/CSS is 80% of the work, PHP or Liquid the rest.

Technical Writer (web docs)

Writes documentation for developer tools. Understanding HTML/CSS lets you write clear code examples and style your own docs site.

QA / Accessibility Tester

Audits sites for WCAG compliance. Knowing semantic HTML and how screen readers parse it is a superpower.

📚 What to learn next, after the certs

  • JavaScript fundamentals — without it, every page is static. Start with W3Schools or freeCodeCamp.
  • A CSS framework — Tailwind CSS is the modern default. Bootstrap is still common in enterprise. Either teaches you patterns.
  • Git + GitHub — every dev job assumes you can clone, branch, commit, push, pull-request. Free, learn-it-once.
  • A JS framework — React first (largest job market), Vue (cleanest for beginners), Svelte (most enjoyable).
  • Accessibility (WCAG 2.1 AA) — legally required for many sites. 🔗 webaim.org has the best free training.
  • Performance & Core Web Vitals — LCP, FID, CLS — Google\'s measurable metrics that affect SEO rankings.

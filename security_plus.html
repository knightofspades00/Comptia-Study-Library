🛡️ 1.0 — General Security Concepts 12%

▼

Core Security Principles

  • CIA Triad: Confidentiality (prevent unauthorized disclosure), Integrity (prevent unauthorized modification), Availability (ensure authorized access when needed)
  • Non-repudiation: Cannot deny performing an action. Achieved via digital signatures
  • AAA: Authentication (verify identity), Authorization (define what they can do), Accounting (log what they did)
  • Least Privilege: Grant only minimum permissions needed to perform a job
  • Defense in Depth: Multiple overlapping security layers — no single point of failure
  • Zero Trust: Never trust, always verify. No implicit trust based on network location

Cryptography Fundamentals

Key Algorithms

  • AES: Advanced Encryption Standard. Symmetric. 128/192/256-bit keys. Current standard for bulk encryption
  • RSA: Asymmetric. 2048+ bit keys recommended. Used for key exchange and digital signatures
  • SHA-256/512: Secure Hash Algorithm. Produces 256 or 512-bit hashes. Replaces MD5 and SHA-1
  • ECC: Elliptic Curve Cryptography. Smaller keys, same security as RSA. Used in mobile/IoT
  • Diffie-Hellman: Key exchange protocol. Allows two parties to establish shared secret over unsecured channel

PKI (Public Key Infrastructure)

  • Certificate Authority (CA): Trusted entity that issues digital certificates
  • Digital Certificate: Contains public key + identity info, signed by CA. Verified via chain of trust
  • CRL: Certificate Revocation List — list of revoked certs
  • OCSP: Online Certificate Status Protocol — real-time cert validation (faster than CRL)

Attack Types

  • SQL Injection: Malicious SQL code injected into input fields to manipulate database queries. Mitigation: parameterized queries, input validation
  • XSS (Cross-Site Scripting): Injects malicious scripts into web pages viewed by others. Reflected (immediate), stored (persistent), DOM-based
  • CSRF: Cross-Site Request Forgery — tricks user's browser into making unintended requests. Mitigation: CSRF tokens
  • Buffer Overflow: Writing data beyond a buffer's boundary to overwrite memory. Can lead to arbitrary code execution
  • Man-in-the-Middle (MITM): Attacker intercepts communication between two parties
  • Replay Attack: Captured authentication data is retransmitted. Mitigated by timestamps and nonces
  • Pass the Hash: Captured hashed credentials used directly for authentication without cracking
  • Credential Stuffing: Automated use of stolen username/password pairs from breaches

Vulnerability Management

  • CVE: Common Vulnerabilities and Exposures — unique ID for known vulnerabilities
  • CVSS: Common Vulnerability Scoring System — 0-10 severity score
  • Patch management: Regular, tested deployment of security patches
  • Vulnerability scanning: Automated identification of weaknesses (Nessus, Qualys)
  • Penetration testing: Authorized simulated attack to find real exploitable vulnerabilities
  • Zero-day: Vulnerability exploited before vendor has issued a patch
🔴 Exam tip: Know the difference between vulnerability scanning (passive, finding weaknesses) and penetration testing (active, exploiting weaknesses).

Network Security Architecture

  • DMZ: Demilitarized Zone — semi-trusted zone for public-facing servers (web, email, DNS) between two firewalls
  • Segmentation: Divide network into zones (VLAN, subnets, firewalls) to limit lateral movement
  • Air gap: Physical isolation — no network connection. Used for highly classified systems
  • Honeypot: Decoy system designed to attract and study attackers
  • Honeynet: Network of honeypots

Cloud Security

  • Shared Responsibility Model: Cloud provider secures infrastructure; customer secures data, identities, and apps
  • CASB: Cloud Access Security Broker — enforces security policies for cloud services
  • CSPM: Cloud Security Posture Management — identifies misconfigurations
  • CWPP: Cloud Workload Protection Platform — secures workloads (VMs, containers)
  • FaaS security: Function-as-a-Service (serverless) — ephemeral, monitor permissions and secrets management

⚙️ 4.0 — Security Operations 28%

▼

Identity & Access Management (IAM)

  • MFA factors: Something you know (password), something you have (token/phone), something you are (biometric). TOTP (Time-based OTP), HOTP (HMAC-based OTP)
  • SSO: Single Sign-On — authenticate once, access multiple resources. Uses SAML, OAuth 2.0, OpenID Connect
  • PAM: Privileged Access Management — controls and monitors admin/root access
  • Directory Services: LDAP (Lightweight Directory Access Protocol) — stores and queries user/group data. Active Directory uses LDAP
  • Federation: Trust relationship between organizations (e.g., login with Google/Microsoft)

Security Tools & Technologies

  • SIEM: Security Information and Event Management — aggregates and correlates logs from multiple sources for threat detection
  • SOAR: Security Orchestration, Automation and Response — automates incident response workflows
  • EDR: Endpoint Detection and Response — continuous monitoring and response on endpoints
  • XDR: Extended Detection and Response — extends EDR across network, cloud, email
  • DLP: Data Loss Prevention — prevents unauthorized data exfiltration
  • IDS/IPS: Intrusion Detection/Prevention System. NIDS = network-based, HIDS = host-based. IDS alerts, IPS blocks

Incident Response

  1. Preparation: Build IR plan, train team, deploy tools
  2. Identification: Detect and analyze the incident
  3. Containment: Limit scope (short-term: isolate; long-term: patch)
  4. Eradication: Remove malware, close vulnerabilities
  5. Recovery: Restore systems, validate functionality
  6. Lessons Learned: Post-incident review, update policies and procedures

Digital Forensics

  • Order of Volatility: CPU registers/cache → RAM → Swap/page file → Hard drive → Remote logs → Backups. Collect most volatile first
  • Chain of custody: Document who handled evidence and when. Required for legal proceedings
  • Legal hold: Suspend normal data deletion when litigation is anticipated

Risk Management

  • Risk = Threat × Vulnerability × Impact
  • Risk responses: Accept (tolerate), Avoid (eliminate activity), Transfer (insurance, outsource), Mitigate (reduce likelihood/impact)
  • Quantitative risk: Dollar values. ALE = SLE × ARO (Annual Loss Expectancy = Single Loss Expectancy × Annual Rate of Occurrence)
  • Qualitative risk: Subjective ratings (high/medium/low)
  • BIA: Business Impact Analysis — identifies critical processes and recovery priorities
  • RTO: Recovery Time Objective — maximum acceptable downtime
  • RPO: Recovery Point Objective — maximum acceptable data loss (time)

Secure Coding Practices

  • Input validation: Validate all user input on the server side. Never trust client-side validation alone.
  • Parameterized queries: Use prepared statements — never concatenate user input into SQL queries.
  • Output encoding: Encode output before rendering in HTML to prevent XSS.
  • CSRF tokens: Unique, session-bound tokens in forms — server verifies they match before processing.
  • Error handling: Never expose stack traces or internal details to end users. Log internally.
  • Code signing: Digitally sign executables and scripts — ensures they haven't been tampered with.
  • SBOM (Software Bill of Materials): Inventory of all software components and their versions — enables rapid response to new CVEs.
  • Static Application Security Testing (SAST): Analyzes source code for vulnerabilities before execution.
  • Dynamic Application Security Testing (DAST): Tests running application for vulnerabilities (fuzzing, automated scanning).

Federation & SSO Protocols

  • SAML 2.0: XML-based. Exchanges authentication assertions between Identity Provider (IdP) and Service Provider (SP). Used for enterprise SSO (e.g., login to Salesforce with your corporate AD credentials).
  • OAuth 2.0: Authorization framework (NOT authentication). Grants third-party apps limited access to resources without sharing credentials. Example: "Allow app to access your Google Drive." Uses tokens.
  • OpenID Connect (OIDC): Authentication layer built on top of OAuth 2.0. Adds ID tokens (JWT) containing user identity info. Used for "Sign in with Google/Microsoft/Apple."
  • JWT (JSON Web Token): Signed (and optionally encrypted) token containing claims. Base64-encoded JSON. Three parts: Header.Payload.Signature. Signature verifies integrity and authenticity.

Privileged Access

  • PAM (Privileged Access Management): Controls, monitors, and audits admin/root access. Features: just-in-time access, session recording, credential vaulting, approval workflows. Examples: CyberArk, BeyondTrust.
  • JIT (Just-in-Time) Access: Grant elevated access only when needed for a specific task, for a limited time. Reduces standing privilege exposure.
  • Break-glass accounts: Emergency admin accounts with extra scrutiny — used only for emergencies. Heavily monitored and audited.
  • Separation of Duties: No single person should be able to complete a critical task alone. E.g., developer cannot deploy their own code to production.
  • Mandatory Vacation / Job Rotation: Forces handoff of duties — detects fraud and hidden malicious activity.

🔒 Endpoint Security & Hardening

▼

Endpoint Hardening Techniques

  • Patch management: Apply OS and application patches regularly. Critical/zero-day patches should be emergency-deployed.
  • Disable unnecessary services: Reduce attack surface by stopping services not needed (Telnet, FTP, NetBIOS).
  • Application whitelisting: Only approved applications can run. AppLocker (Windows Enterprise) enforces this. Blocks unknown malware even if AV misses it.
  • Host-based firewall: Windows Defender Firewall / iptables. Block inbound/outbound by port, protocol, IP.
  • Full disk encryption: BitLocker (Windows), FileVault (macOS), LUKS (Linux). Protects data if device is stolen.
  • BIOS/UEFI password: Prevents unauthorized boot from external media.
  • Secure Boot: UEFI feature verifying boot loader and kernel signatures. Prevents rootkits from loading before OS.
  • TPM (Trusted Platform Module): Hardware chip that stores keys, measurements. Required for BitLocker and Windows 11.
  • Baseline configuration: Standard hardened configuration applied to all systems. Deviations flagged by compliance tools.
  • CIS Benchmarks: Center for Internet Security best-practice configuration guides for OS, applications, and cloud. Industry standard for hardening.

EDR vs Traditional AV

  • Traditional AV: Signature-based — identifies known malware by comparing file hashes/patterns. Blind to novel/zero-day attacks and fileless malware.
  • EDR: Monitors endpoint behavior continuously. Detects anomalies (unusual process spawning, lateral movement, memory injection). Can respond: isolate endpoint, kill process, roll back changes.
  • XDR: Extends EDR across network, email, cloud. Correlates telemetry from multiple sources for better detection and investigation.
  • MDR: Managed Detection and Response — outsourced SOC team monitoring EDR/XDR on your behalf.

Mobile Device Security

  • MDM vs MAM: MDM manages the entire device. MAM (Mobile Application Management) manages only corporate apps — better for BYOD.
  • Containerization: Isolates corporate data/apps in encrypted container on personal device. Corporate data can be wiped without touching personal data.
  • Jailbreaking / Rooting: Removes OS security restrictions. MDM can detect and block jailbroken devices from accessing corporate resources.
  • Certificate-based authentication: Deploy certificates to mobile devices for Wi-Fi/VPN auth instead of passwords — more secure, no credential theft risk.

Cloud Security Controls

  • CASB (Cloud Access Security Broker): Sits between users and cloud services. Enforces policies: DLP, access control, threat protection, compliance. Provides visibility into shadow IT (unauthorized cloud apps).
  • CSPM (Cloud Security Posture Management): Continuously scans cloud environment for misconfigurations (public S3 buckets, overly permissive IAM roles, unencrypted databases). Auto-remediation available.
  • CIEM (Cloud Infrastructure Entitlement Management): Manages and right-sizes cloud IAM permissions. Identifies excessive permissions and enforces least privilege in cloud.
  • Cloud security groups: Virtual firewall applied to cloud instances (stateful). Controls inbound/outbound traffic at instance level.
  • NACLs (Network ACLs): Subnet-level stateless firewall in cloud VPCs. Applied before security groups.
  • Key Management Service (KMS): Managed service for creating, rotating, and controlling encryption keys. Separates key management from data storage — keys can be customer-managed (BYOK).

Cloud Threats

  • Misconfigured storage: Public S3 buckets, Azure Blob containers — most common source of cloud data breaches. Always check bucket ACLs/policies.
  • Overprivileged IAM: Service accounts or users with excessive permissions. Apply least privilege; audit IAM quarterly.
  • Insecure APIs: Cloud resources exposed via APIs without proper auth. Enforce API keys, OAuth, rate limiting.
  • Side-channel attacks: Shared compute resources (VMs on same host) — theoretical risk of data leakage between VMs via CPU cache timing attacks.

Network Security Zones

  • Intranet: Internal corporate network — trusted zone
  • Extranet: Network extended to trusted third parties (partners, suppliers) via VPN or dedicated link
  • DMZ: Semi-trusted — public-facing servers. Two firewalls required for true DMZ
  • Guest network: Isolated network for visitors. No access to internal resources. Implemented via separate VLAN + firewall rules

Data Loss Prevention (DLP)

  • Cloud DLP: Scans cloud storage (OneDrive, Google Drive) for sensitive data violations
  • Data classification + labeling: Tag data (Public, Internal, Confidential, Restricted) and apply DLP policies per classification

Activity 3: Zero Trust Architecture

Zero Trust is a critical modern security architecture concept heavily featured on SY0-701. Read the following description and answer the comprehension questions.

Zero Trust (ZT) is a security model built on the principle of "never trust, always verify." Traditional perimeter security assumed that anything inside the network was trustworthy. Zero Trust eliminates this assumption — every user, device, and application must authenticate and be authorized before accessing any resource, regardless of whether they're inside or outside the corporate network. Core pillars include: strong identity verification (MFA), device health validation, least-privilege access, microsegmentation, and continuous monitoring with analytics.

🔐 The CIA Triad — Core Security Principles

Every security control, policy, and threat maps back to one or more of the three pillars: Confidentiality, Integrity, and Availability. Understanding which pillar is at risk in a scenario is key to answering Security+ questions.

Example — Confidentiality attack: A hacker intercepts unencrypted email and reads private messages (eavesdropping). Fix: use TLS encryption.
Example — Integrity attack: An attacker modifies a software download so malware is included. Fix: verify SHA-256 hash before installing.
Example — Availability attack: A DDoS flood overwhelms a web server, taking it offline. Fix: use CDN, rate limiting, and failover infrastructure.

🚨 Incident Response Lifecycle (NIST SP 800-61)

NIST defines four phases for responding to a security incident. Security+ questions will ask you to identify the correct phase for a given action (e.g., "which phase involves preserving evidence?").

Exam Tip
Evidence preservation (chain of custody, disk imaging) happens during Detection & Analysis — before containment — so it is not accidentally destroyed.

⚔️ The Cyber Kill Chain (Lockheed Martin)

The Kill Chain describes the stages an attacker moves through from initial research to achieving their objective. Defenders try to interrupt the chain as early as possible — each stage is a detection opportunity.

Exam Tip
Email filtering and user training stop the chain at Delivery. Patching stops it at Exploitation. IDS/EDR stops it at Installation. Blocking suspicious outbound stops it at C2.

📊 Risk Calculations

Security+ includes quantitative risk math. Learn these four formulas — they appear directly in exam questions.

TermFormula / DefinitionExample
Asset Value (AV)What the asset is worth in dollarsServer worth $50,000 → AV = $50,000
Exposure Factor (EF)% of asset lost in one incident (0–100%)Fire destroys 40% of server room → EF = 0.40
Single Loss Expectancy (SLE)AV × EF$50,000 × 0.40 = $20,000 per incident
Annualized Rate of Occurrence (ARO)Expected incidents per yearFire risk estimated at 0.1/year (once per decade)
Annualized Loss Expectancy (ALE)SLE × ARO$20,000 × 0.1 = $2,000/year expected loss
Decision Rule: If a security control costs less than the ALE, it is economically justified. A $1,500/year fire suppression contract is worth it ($1,500 < $2,000 ALE).

"Symmetric = Same key (like a padlock) · Asymmetric = Asymm-key (public + private pair)"

"Phishing = fishing with a fake lure. Vishing = Voice phishing. Smishing = SMS phishing."

"Hashing is like grinding coffee — you can't ungrind it. Same input ALWAYS = same hash."
MD5: 128-bit (weak, broken — still on exam) · SHA-1: 160-bit (deprecated) · SHA-256/512: strong, used for file integrity and certificates · Salting adds random data before hashing to defeat rainbow tables

🗣️ CompTIA Question Language

CompTIA uses specific trigger words that change the right answer. Learn to spot them:

  • "MOST likely" — eliminate impossible answers; pick the best fit
  • "BEST" — there may be 2 correct answers; one is more efficient/secure
  • "FIRST" — follow the correct order/process (e.g., troubleshooting steps)
  • "LEAST" / "EXCEPT" — you're looking for the wrong one
  • "ALWAYS" / "NEVER" — these are usually wrong; tech rarely works in absolutes

✂️ Elimination Technique

  1. Read the question stem carefully — what is it really asking?
  2. Cross out any answer that is obviously wrong or out of scope
  3. If 2 answers look similar, one of them is usually correct
  4. Look for answers that are "too extreme" — usually distractors
  5. If completely unsure, pick the more conservative/secure option

  • Most PBQs test basic knowledge in a visual way
  • Flagging and returning with fresh eyes often helps

🗺️ 1.0 — Geographic Data Models

📚 Pro docs ▼

What is GIS? — the layered worldview

  • GIS = software that maps and analyzes geographic information. Every layer is data tagged with a location.
  • Three pieces in every GIS dataset: geometry (where), attributes (what), spatial reference (how to align). Missing any one and the map is broken.
  • Layers stack bottom-up: basemap underneath, operational data on top. Drawing order matters — points on top of polygons, labels on top of everything.
  • ArcGIS Pro is the desktop authoring app; ArcGIS Online (AGOL) is the cloud sharing/serving platform. You build in Pro, publish to AGOL, share to your audience.
  • Common verbs: capture (digitize, GPS, import), store (geodatabase), analyze (geoprocessing), visualize (cartography), share (web maps, dashboards).

Vector data — features represented by shapes

  • Points — discrete single locations (a school, a fire hydrant, a sample site). One x,y pair per feature.
  • Lines (polylines) — connected vertex chains (a road, a river, a power line).
  • Polygons — closed areas (a state, a parcel, a fire perimeter). First vertex repeats as last; counter-clockwise rings = outer boundary, clockwise rings = hole.
  • Multipoint / multilinestring / multipolygon — one feature row can hold multiple disjoint geometries (e.g., the State of Hawaii = one feature, many island polygons).
  • Vector is best for things with sharp boundaries — buildings, roads, parcels. Scale-independent: still crisp when zoomed in.

Raster data — the world as a grid

  • Cells (pixels) with a single value each — elevation, temperature, land cover, satellite reflectance.
  • Cell size = ground distance one pixel represents (10 m, 30 m, 1 m). Trade-off: smaller cells = sharper detail and a much larger file.
  • Bands: a raster can have one band (a DEM) or many (4-band orthoimagery = Red, Green, Blue, Near-Infrared).
  • Continuous rasters (elevation, temperature) gradient symbology; discrete/thematic rasters (land-cover classes) use unique values.
  • Best for phenomena that change smoothly across space — terrain, climate, imagery — and for analyses like slope, viewshed, hillshade, NDVI.

Attribute tables — the data behind the features

  • Every feature class has a table: one row per feature, columns are fields (attributes).
  • OBJECTID — automatic unique row ID. Don't edit or delete; ArcGIS uses it internally for joins and selections.
  • Shape (SHAPE) — the geometry column. Symbology and spatial analysis read this column.
  • Field data types: Short Integer, Long Integer, Double, Text, Date. Pick once; changing later requires reloading or recreating the field.
  • Joins attach another table's attributes by a common key (census tract data → census tract polygons via GEOID). Joins are temporary until you export to a new feature class.

Common file formats you'll see

  • Shapefile (.shp + sidecars) — Esri's legacy format, ubiquitous. Limits: 2 GB max, 10-character field names, no native time data. Always travels as a folder of 4-6 files: .shp .shx .dbf .prj at minimum.
  • Feature class in a geodatabase — the modern Esri vector container. No 2 GB cap, full field names, supports domains, subtypes, attribute rules.
  • GeoJSON — open web standard, plain text JSON. Spec mandates WGS 84 (EPSG 4326).
  • KML / KMZ — Google Earth's format, good for 3D globe viewing. KMZ = zipped KML.
  • CSV with X/Y — text rows with lat/long columns; import via XY Table to Point.

🧭 2.0 — Coordinate Systems & Projections

📚 Pro docs ▼

The Earth's shape — datums, ellipsoids, geoids

  • The Earth is almost a sphere — slightly flattened at the poles. We approximate it with an ellipsoid (a mathematical model with semi-major and semi-minor axes).
  • A datum ties the ellipsoid to specific reference points on the Earth. Same coordinates on different datums = different real-world places.
  • The geoid is the actual gravitational "sea level" surface — used for elevation (height above sea level). Bumpy, not smooth.
  • Why it matters: a point's coordinates depend on which datum is assumed. "30.0, 45.0" on WGS 84 vs NAD 27 can differ by hundreds of meters.

Geographic vs Projected CRS (GCS vs PCS)

  • Geographic CRS (GCS) uses angles — latitude/longitude on the round Earth. Units = degrees. Distances and areas are NOT directly measurable (a degree of longitude shrinks with latitude).
  • Projected CRS (PCS) uses Cartesian X/Y on a flat plane. Units = meters or feet. Distances and areas are measurable. Trade-off: every projection distorts something.
  • Rule of thumb: store and exchange in a GCS; analyze and measure in a PCS.
  • ArcGIS Pro projects layers on-the-fly to match the map frame's CRS, but analysis tools work most reliably when every input shares one PCS.

Common datums you'll meet

  • WGS 84 — global GPS standard. Default for GeoJSON, KML, web maps, almost all GPS receivers.
  • NAD 83 — modern North American datum. Used for State Plane and most US authoritative data.
  • NAD 27 — older North American datum. Still common in historical/oil-and-gas data; can differ from NAD 83 by ~100 m in the lower 48.
  • Mixing datums without a transformation is the #1 source of "my layers don't line up" tickets. ArcGIS Pro warns you and asks which datum transformation to apply.

Projection families — what each preserves

  • Conformal — preserves local angles/shape. Examples: Mercator, Lambert Conformal Conic. Best for navigation, weather.
  • Equal-area — preserves area. Examples: Albers Equal Area, Mollweide. Best for population maps, choropleths.
  • Equidistant — preserves distance from one or two reference points. Good for "distance from here" analyses.
  • Compromise — preserves nothing perfectly but minimizes overall distortion. Examples: Robinson, Winkel Tripel — common general-purpose world maps.
  • No projection preserves all properties. Pick by what your map must communicate truthfully.

Projections you'll meet most often

  • Web Mercator (EPSG 3857) — the web-map default (Google, OSM, AGOL basemaps). Conformal but distorts area badly toward the poles — Greenland looks the size of Africa; it isn't.
  • UTM — 60 zones, each 6° wide. Within a zone, distances are accurate to roughly 1 part in 2500. Used by surveyors, ecologists, the US military.
  • State Plane (NAD 83) — US state-specific zones designed for high accuracy within each state. East-west states (e.g., Texas) use Lambert; north-south states (e.g., Idaho) use Transverse Mercator.
  • Albers Equal Area Conic — the standard "lower-48" choropleth projection.

EPSG codes — the universal CRS shorthand

  • Every CRS has an EPSG code — a number you copy/paste into any GIS software to specify the CRS unambiguously.
  • WGS 84 = 4326 · Web Mercator = 3857 · NAD 83 = 4269 · NAD 27 = 4267.
  • Look up unknown codes at epsg.io.
  • In ArcGIS Pro, every CRS dialog shows the WKID (Well-Known ID) — usually the EPSG code.

Project vs Define Projection — the most-tested distinction

  • Project (or Project Raster) — converts data from one CRS to another. Geometry coordinates change. Use when you actually need to re-project.
  • Define Projection — sets the CRS label on data that has no label or the wrong label. Geometry coordinates do NOT change. Use when a dataset came in unlabeled.
  • Wrong tool = silent disaster. If you "Define" data with the wrong CRS, the numbers stay but the label is a lie — every subsequent operation projects to wrong real-world locations and you may never notice.

🎨 3.0 — Cartography & Symbology

📚 Pro docs ▼

The five must-have map elements

  • Title — what the map is about, at the top, sized for the audience.
  • Legend — explains every symbol. Match the in-map symbol exactly; group related entries; drop layers that don't need explaining.
  • Scale bar — graphic, resizes correctly if printed at different sizes. Avoid representative-fraction text on web maps that re-scale.
  • North arrow — required unless north is clearly up and obvious (small inset maps can skip).
  • Neat line + credits/source/date — frame the map, cite the data source, note the projection. Professionally required.

Symbology approaches in ArcGIS Pro

  • Single Symbol — every feature drawn identically. Use for context layers.
  • Unique Values (categorized) — one symbol per discrete category (land-cover class, school type). Up to ~7 categories before the legend becomes unreadable.
  • Graduated Colors — color ramp keyed to a numeric value (income, density). Pick a classification method (next).
  • Graduated Symbols — point/line/polygon size keyed to a numeric value. Use when color is already taken.
  • Heatmap — density estimation for point data. Great for "where's the cluster?" but bad for comparing exact counts.

Classification methods for graduated symbology

  • Equal Interval — splits the value range into equal-width bins. Easy to read; bad when data is skewed (most features land in one bin).
  • Quantile — equal feature count per bin. Highlights ranking, but can put very different values in the same bin.
  • Natural Breaks (Jenks) — algorithm finds gaps in the data and breaks there. Best general default for skewed data.
  • Standard Deviation — bins relative to the mean. Use when "above/below average" matters most.
  • Manual — when the data has meaningful real-world thresholds (e.g., flood-stage levels, IRS tax brackets).

Color — the most powerful and most-misused tool

  • Sequential ramps (light → dark of one hue) — for ordered data: low to high (income, density, elevation).
  • Diverging ramps (two hues meeting at a neutral middle) — for data with a meaningful midpoint (gain/loss, above/below average).
  • Qualitative palettes (distinct hues, no order) — for categorical data (land-cover types, political party).
  • Color-blind safe: stick to ColorBrewer ramps (built into ArcGIS Pro). Avoid red-green diverging without testing.
  • Less is more: too many hues compete. Pick one ramp per layer and reserve bright colors for the focus of the map.

Labels, annotation, and visual hierarchy

  • Labels — generated dynamically from a field. Reposition as the map redraws; toggle per-scale ranges. Light to maintain.
  • Annotation — fixed text features stored in the geodatabase. Hand-place when labels can't be positioned correctly automatically.
  • Visual hierarchy: the important stuff is bigger, brighter, or sharper. The map should answer "what is this about?" within one second of looking.
  • Standard layout: title at the top, legend bottom-right, scale bar bottom-left, north arrow bottom-right corner. Don't crowd the map frame.

🔬 4.0 — Spatial Analysis & Geoprocessing

📚 Pro docs ▼

The geoprocessing model

  • A tool takes input data + parameters and produces output data. Always file-based — outputs are new datasets; originals are untouched.
  • Parameters — required and optional. ArcGIS Pro shows required ones with a red asterisk.
  • Environments — defaults that apply to every run: output CRS, processing extent, raster cell size, mask. Set per project to avoid surprises.
  • History — every geoprocessing run is logged. Right-click → Copy Python Snippet to script later.

Vector overlay operations

  • Buffer — the area within distance D of a feature. Inputs: features + distance + optional dissolve to merge overlapping buffers.
  • Clip — cookie-cutter: keep features (or parts) INSIDE another layer. Attributes stay; geometry is trimmed.
  • Intersect — features where two layers overlap. Output carries attributes from BOTH inputs.
  • Union — output covers every part of BOTH layers (overlap + non-overlap parts), each tagged with which input it came from.
  • Erase — remove area in B from A. Symmetric Difference — "parts that don't overlap" (XOR of A and B).

Proximity & selection

  • Near / Generate Near Table — find the nearest feature in another layer for each input feature; record the distance.
  • Select By Attributes — SQL on the attribute table. Builds a query like POPULATION > 50000 AND STATE = 'TX'.
  • Select By Location — spatial relationships: intersects, within, contains, within a distance of. Combine with Select By Attributes for compound queries.
  • Spatial Join — bring attributes from B into A by spatial relationship. One-to-one (default with summary stats) or one-to-many (each spatial match becomes a row).

Raster analysis essentials

  • Map Algebra — arithmetic on raster values, cell by cell: NDVI = (NIR - Red) / (NIR + Red).
  • Reclassify — assign new values to ranges. Convert continuous → discrete (elevation → high/medium/low).
  • Slope — degree or percent rise from a DEM. Aspect — compass direction the slope faces (0–360°, with -1 = flat).
  • Hillshade — illuminate terrain from a chosen sun angle/azimuth to make 3D shape readable.
  • Zonal Statistics — compute stats (mean, max, count) of one raster within zones defined by another (e.g., mean elevation per county).

Geoprocessing environments — the gotchas

  • Output Coordinate System — if inputs differ in CRS, set this explicitly. Otherwise tools may pick the first input's CRS.
  • Extent — limit processing to a rectangle. Speeds large-area analyses dramatically.
  • Cell Size (rasters) — defaults to the largest input's cell size. Override to control output resolution.
  • Mask — only process cells inside this polygon/raster. Common pattern: run analysis nationally, mask to one state for the deliverable.

ModelBuilder vs Python (ArcPy)

  • ModelBuilder — visual flow: input → tool → output → next tool. Great for documenting analyses and sharing workflows with non-coders.
  • Python (ArcPy) — script the same operations. Better for loops, branches, integration with non-Esri tools.
  • Both produce the same results. Pick by audience: ModelBuilder for the team meeting, Python for the scheduled job.

🌐 5.0 — ArcGIS Online & Web GIS

📚 AGOL docs ▼

AGOL accounts, orgs, and roles

  • ArcGIS Online (AGOL) is Esri's cloud platform. Your school's organization has its own URL — typically yourschool.maps.arcgis.com.
  • Roles: Viewer (read-only), User (create/own content), Publisher (publish hosted services), Administrator (manage the org).
  • Credits — Esri's billing unit. Some operations consume credits: geocoding, storage of hosted layers, premium content access.
  • My Content — items you own. My Groups — collections you've joined or created.

Content types you'll create

  • Web Map — a basemap + operational layers + popups + bookmarks. The most common deliverable.
  • Web App — a web map wrapped in a user interface (Dashboard, Experience Builder, Map Viewer).
  • Hosted Feature Layer — vector data hosted by AGOL. Editable from the web, queryable via the REST API.
  • Hosted Tile Layer — raster tiles pre-rendered at fixed zoom levels. Fast but not editable.
  • Hosted Vector Tile Layer — vector tiles styled at the client. Crisp at any zoom, restyleable in the browser.

Sharing — the gate for who sees what

  • Owner — only you (default for new items).
  • Organization — anyone signed into your org. Good for in-class sharing.
  • Groups — share with named groups (a class period, a project team).
  • Everyone (public) — anyone on the internet. Required for public-facing maps and apps.
  • An app's audience can only be as wide as the items inside it. If the map is "Org" and the app is "Public," external viewers see broken layers — share every layer to match the widest audience.

Building a web map — the workflow

  • 1. Pick a basemap (Imagery, Streets, Topographic, Light Gray Canvas).
  • 2. Add operational layers from My Content, Living Atlas, or a URL.
  • 3. Style each layer with Smart Mapping — AGOL inspects the data and suggests defensible symbology.
  • 4. Configure popups — choose which fields show, rename headers, add Arcade expressions for derived fields.
  • 5. Save with full item details, then share.

ArcGIS apps you should know

  • Dashboards — KPI cards + charts + a map driven by a single feature layer. Near-real-time monitoring.
  • Survey123 — smart forms that collect into a feature layer. Field-data collection.
  • Field Maps — mobile app for navigation, data collection, inspections. Offline-capable.
  • Experience Builder — drag-and-drop web-app builder. Replaced the older Web AppBuilder.
  • StoryMaps — narrative + maps + photos. Best for explaining a project to a non-GIS audience.

Publishing from ArcGIS Pro → AGOL

  • Sign into your org in Pro: Project tab → Portals → set active portal.
  • Right-click a layer → Share As Web Layer. Choose Feature, Tile, or Vector Tile.
  • Set the sharing level, tags, and item details BEFORE you click Publish — easier than retrofitting after.
  • The published item lands in My Content in AGOL. Now add it to a web map.

Smart Mapping & item-details best practices

  • Smart Mapping inspects your field data type and distribution and suggests defensible defaults. Use it as the starting point for choropleths and graduated styles.
  • Always fill out item details: Title, Summary, Description, Terms of Use, Credits, Tags, Thumbnail. Discoverable items get used; bare items collect dust.

🗄️ 6.0 — Geodatabases, Metadata & Sharing

📚 Pro docs ▼

Geodatabase types

  • File geodatabase (.gdb) — a folder of binary files. Default for most ArcGIS Pro projects. Supports up to 1 TB per feature class; domains, subtypes, attribute rules.
  • Mobile geodatabase (.geodatabase) — a single SQLite file. Easy to share, embed in mobile apps, version with git.
  • Enterprise geodatabase — runs inside SQL Server, Oracle, PostgreSQL, etc. Multi-user editing, versioning, replication. Used in large orgs.
  • Shapefile is NOT a geodatabase — it's a single-feature-class legacy container with strict limits.

Schema concepts

  • Feature dataset — a container inside a geodatabase that groups feature classes sharing a CRS. Required for topology and some networks.
  • Feature class — one geometry type + one CRS + an attribute table. Lives inside a geodatabase (or a feature dataset).
  • Domains — controlled lists of allowed values for a field (e.g., a domain "RoadType" with values Highway/Arterial/Local). Prevents typos and standardizes categories.
  • Subtypes — different feature behaviors within one feature class (Roads with subtypes Highway/Arterial — each gets its own default attribute values and symbology).
  • Attribute rules — Arcade expressions that fire on insert/update/delete to validate or auto-populate fields.

Topology and relationship classes (intro)

  • Topology — rules that enforce spatial relationships (parcels must not overlap, roads must connect at intersections). The Error Inspector flags violations for correction.
  • Relationship class — links rows in one table to rows in another by a key (one parcel → many tax records). Enforces referential integrity.

Metadata — the part everyone skips

  • Metadata = data about your data: title, abstract, lineage, source, contact, CRS, fields, dates.
  • Standards: ISO 19115 (international), FGDC CSDGM (US legacy). ArcGIS Pro supports both via the Metadata style setting.
  • Why it matters: without metadata, your shapefile from 2017 is mystery data. With it, the next person knows what they have and how to use it correctly.
  • Edit metadata in Pro: right-click a dataset → Edit Metadata.

Sharing best practices & export formats

  • Before sharing, complete Item Details: Title (clear), Summary (1–2 sentences), Description (full context), Terms of Use, Credits.
  • Tags drive search — include the topic, the area, and the dataset type.
  • Export formats: Shapefile (universal but limited), file geodatabase (best for ArcGIS users), GeoJSON (web), KML (Google Earth), CSV with WKT or X/Y (spreadsheets).
  • If sharing publicly, double-check no PII or sensitive locations are encoded in attributes or in raw vertex coordinates.
